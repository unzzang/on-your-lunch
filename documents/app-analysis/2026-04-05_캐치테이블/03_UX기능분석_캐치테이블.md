# 캐치테이블(CatchTable) UX/기능 분석

> 분석 일자: 2026-04-05
> KRQ-5: 캐치테이블의 UX 완성도는 어떤 수준이며, 차별화된 UX 패턴이 있는가?

---

## 1. 온보딩

| 항목 | 내용 |
|------|------|
| 로그인 방식 | 휴대폰 번호, 카카오, 네이버, Apple (4가지) |
| 소셜 로그인 최적화 | 카카오/네이버 계정 정보 자동 수집, 별도 입력 불요 |
| 회원가입 단계 | 소셜 가입 시: 동의 → 자동 수집 → 완료 (2~3단계) |
| 가입 소요 시간 | 평균 1,804.5초 (개선 전 2,429.4초 대비 26% 감소) |
| 가입 완료율 | 24.5% (개선 전 21.7% 대비 2.8%p 증가) |
| 사용자 액션 수 | 10회 (개선 전 14회에서 감소) |
| 가입 VOC | 월 45.3건 → 9건 (80% 감소) |

출처: [캐치테이블 Medium - 회원가입 개선기](https://medium.com/catchtable/%ED%9A%8C%EC%9B%90%EA%B0%80%EC%9E%85-%EB%8D%94-%EC%89%BD%EA%B2%8C%ED%95%A0-%EC%88%98-%EC%97%86%EC%9D%84%EA%B9%8C-6888f93b8ec8)

- [사실] 데이터 기반의 체계적인 온보딩 최적화를 수행. 필드 자동 포커싱, 버튼 레이블 개선("인증번호 요청" → "인증번호 받기") 등.
- [해석] 완료율 2.8%p 상승은 MAU 500만 규모에서 월 14만 명 추가 전환에 해당.
- [시사점] 온보딩 퍼널 최적화의 ROI가 매우 높음을 보여주는 사례.

## 2. IA(정보 구조) — 5탭 구성

| 탭 | 명칭 | 핵심 기능 |
|----|------|----------|
| 1 | 홈 | 검색바, 주제별 큐레이션, 추천 식당, 캐러셀 콘텐츠 |
| 2 | 검색하기 | 지역/음식/가격/테이블타입/분위기/편의시설 필터 |
| 3 | 타임라인 | 커뮤니티, 리뷰 및 미식 정보 공유 |
| 4 | 마이다이닝 | 예약 내역, 방문 식당, 웨이팅 상태 |
| 5 | 마이페이지 | 저장 식당, 작성 리뷰, 프로필 설정 |

출처: [브런치](https://brunch.co.kr/@bydot/1)

**네비게이션 깊이:**
```
홈 ─┬─ 식당 상세 ─── 예약하기 ─── 예약 확인
    ├─ 큐레이션 상세 ─── 식당 목록 ─── 식당 상세
    └─ 주류/밀키트 상세

검색 ─── 검색 결과 ─── 식당 상세 ─── 예약하기

타임라인 ─── 게시물 상세 ─── 식당 상세

마이다이닝 ─┬─ 예약 상세 ─── 리뷰 작성
            └─ 웨이팅 상태
```

전체 화면 수 추정: 약 40~60개

- [해석] "타임라인" 탭으로 커뮤니티/콘텐츠 축을 추가한 것이 차별점. 일반 예약 앱이 4탭인 것에 비해 소셜 요소를 탭 레벨로 격상.
- [시사점] 단순 예약 도구를 넘어 미식 라이프스타일 플랫폼으로의 전환을 의도.

## 3. 핵심 기능 목록

| 카테고리 | 기능 | 설명 |
|---------|------|------|
| **예약** | 실시간 예약 | 24시간 온라인, 날짜/시간/인원 조건 검색 |
| | 빈자리 알림 | 만석 시 빈자리 발생 푸시 알림 |
| | 레스토랑 함께 고르기 | 여러 식당 비교 선택 (소셜) |
| **웨이팅** | 원격 웨이팅 | 온라인 줄서기 |
| | 웨이팅 미루기 | 대기 순서 조정 |
| **결제** | 캐치페이 자동결제 | 앱 등록 카드로 자동 결제 (재사용률 80%+) |
| | 예약금 0원 결제 | 카드 등록만으로 예약 확정 |
| **커뮤니티** | 타임라인 | 미식 리뷰/정보 공유 |
| | 인증 리뷰 | 실제 방문자만 작성 가능 (150만 개+) |
| **커머스** | 주류 배송/픽업 | 와인, 위스키 |
| | 밀키트, 할인 티켓 | 유명 맛집 협업 |
| **검색** | 단어 조합 검색 | "압구정 스시오마카세" 자연어 |
| | 세분화 필터 | 지역/음식/가격/분위기/편의시설 |
| **글로벌** | 다국어 지원 | 20개 언어 |

## 4. 핵심 플로우

```
[1] 앱 설치 → 소셜 로그인 → 약관 동의 → 정보 자동 수집 → 가입 완료
[2] 홈 화면 → 검색 또는 큐레이션 → 식당 검색 (필터 적용)
[3] 식당 상세 → 메뉴/사진/리뷰/위치 확인 → "예약하기"
[4] 캘린더 날짜 선택 → 시간대 → 인원수
[5] 예약 정보 확인 → 결제 → 예약 완료
[6] 카카오톡 확인 알림 → 방문일 리마인드 푸시
[7] 매장 방문 → 식사 → (캐치페이 시) 자동 결제
[8] 방문 완료 → 리뷰 작성 유도 → 예약금 환불
```

출처: [캐치테이블 예약 방법](https://hajungood.com/%EC%BA%90%EC%B9%98%ED%85%8C%EC%9D%B4%EB%B8%94-%EC%96%B4%ED%94%8C%EC%9D%84-%ED%86%B5%ED%95%9C-%EC%8B%9D%EB%8B%B9-%EC%98%88%EC%95%BD-%EB%B0%A9%EB%B2%95-%EC%89%BD%EA%B3%A0-%EB%B9%A0%EB%A5%B4%EA%B2%8C/)

## 5. UI 패턴 및 디자인 시스템

### 비주얼 톤앤매너

| 항목 | 관찰 결과 |
|------|----------|
| 전체 톤 | "호텔 예약 앱 수준의 격식 있는 느낌" |
| 배경 | 화이트 기반, 넉넉한 여백 |
| 색상 계열 | 그레이/베이지 안정감 + 포인트 컬러 (브랜드 블루) |
| 이미지 | 대형 레스토랑 사진 중심, 파인다이닝 분위기 강조 |
| 애니메이션 | 부드러운 페이드인, 세련된 전환 |

출처: [맛집 예약앱 UXUI 비교](https://brunch.co.kr/@4e2b4f97d7214af/60)

### 차별화된 UX 패턴 5가지

1. **"만석 → 빈자리 알림" 전환** — 좌절을 기회로 바꿈. 이탈 방지 + 리인게이지먼트
2. **"레스토랑 함께 고르기"** — 그룹 의사결정 소셜 기능
3. **"웨이팅 미루기"** — 대기 중 순서 조정, 사용자 통제감 강화
4. **"예약금 0원 결제"** — 카드 등록만으로 확정, 전환율 극대화 + 노쇼 방지
5. **단어 조합 검색** — "압구정 스시오마카세" 자연어 처리

## 6. 개인화

| 항목 | 내용 |
|------|------|
| 큐레이션 방식 | 매장 데이터 기반 주제별 큐레이션 |
| 맥락 기반 추천 | 동행자, 식사 목적, 분위기 등 "식사 컨텍스트" 고려 |
| 재구매율 | 1년 내 52.6% |

출처: [브런치](https://brunch.co.kr/@bydot/1), [브런치](https://brunch.co.kr/@seongyeon06/50)

- [해석] 알고리즘 기반 개인화보다는 에디토리얼 큐레이션에 가까운 것으로 보인다. 52.6% 재구매율은 외식 예약 서비스치고 높은 수준.

## 7. 알림 전략

| 유형 | 설명 |
|------|------|
| 빈자리 알림 | 등록 식당에 빈자리 발생 시 푸시 |

## 8. UX 완성도 종합 평가

| 강점 | 약점 |
|------|------|
| 고급스러운 비주얼 톤 | 자동 로그아웃 세션 불안정 |
| 단계별 예약 플로우 | 앱 부팅 속도 이슈 |
| 온보딩 최적화 (26% 시간 감소) | 글로벌 버전 UX 미흡 |
| 빈자리 알림 전환 패턴 | 웨이팅 순서 조정 유연성 부족 |
| 캐치페이 자동결제 (80% 재사용) | 예약금 환불 자동화 미완성 |

**UX 완성도: 높음 (10점 만점 기준 7~8점 추정)**

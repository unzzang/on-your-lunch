# 백엔드 아키텍처 설계

백엔드의 전체 구조를 설계한다. API 설계, DB 스키마, 인증 흐름, 기술 스택 선정을 포함한다.

## 작업 기준

### 설계 순서

1. 기술 스택 선정 (아키텍처 패턴 카탈로그 참조)
2. ERD 설계 (엔티티, 관계, 인덱스)
3. API 설계 (엔드포인트, 인증, 에러 코드)
4. 인프라 구조 (배포, 모니터링)

### 설계 원칙

- 기능 명세서의 모든 요구사항을 ERD와 API에 빠짐없이 반영한다
- 의사결정은 DR로 기록한다

## 2-1. 스키마 ↔ 백엔드 정합성

DB 스키마(Prisma)와 백엔드 코드가 일치하는지 대조한다.

**검증 항목:**

| 검증 | 확인 내용 | 불합격 기준 |
|------|----------|-----------|
| 모델 커버리지 | 스키마의 모든 model에 대응하는 모듈이 존재하는가 | 대응 모듈 없는 model 1개 이상 |
| 필드 반영 | 각 model의 모든 필드가 사용되거나, 미사용 사유가 명시되어 있는가 | 사유 없이 미사용 필드 존재 |
| CRUD 완전성 | 인덱스가 암시하는 조회 패턴에 대응하는 엔드포인트가 있는가 | 조회용 인덱스가 있는데 GET 없음 |
| Enum 일치 | 스키마 enum과 shared-types enum이 동일한가 | 값 불일치 또는 누락 |
| 관계 반영 | 1:N, M:N 관계가 API 응답에 반영되는가 | 관계 정의만 있고 쿼리에서 미사용 |
| 고아 테이블 | 어떤 모듈에서도 참조하지 않는 테이블이 있는가 | 1개 이상 |

**절차:**

1. `schema.prisma`의 model 목록을 추출한다
2. 각 model에 대응하는 모듈의 service·controller·DTO를 읽는다
3. 필드 사용 여부를 확인한다
4. 인덱스가 의도하는 쿼리 패턴을 확인한다
5. 불일치 항목을 심각도(상/중/하)로 분류하여 기록한다

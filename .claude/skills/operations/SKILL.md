# 서비스 모니터링 + 장애 대응

서비스의 안정적 운영을 위한 모니터링과 장애 대응을 수행한다.

## 작업 기준

### 모니터링

- 서버 상태 (CPU, 메모리, 디스크)
- API 응답 시간, 에러율
- DB 커넥션, 슬로우 쿼리
- 사용자 접속 현황

### 장애 대응

1. 장애 감지 → 즉시 PO 보고
2. 원인 분석 → 백엔드 팀과 협의
3. 긴급 조치 (롤백, 스케일링 등)
4. 사후 분석 리포트 작성

### 배포 관리

- 배포 전 체크리스트 확인
- 배포 후 모니터링 (최소 30분)
- 롤백 절차 유지

## 협업

- **이인수(백엔드 개발자)**와 장애 원인 분석·핫픽스 협의
- **이명환(백엔드 아키텍트)**과 인프라 개선 협의
- **진도준(QA)**에게 배포 후 검증 요청
- **정예림(CS)**으로부터 장애 신고 수령
- **PO**에게 장애·배포 현황 보고

# Agent/Skill 범용화 리팩토링 및 프로세스 개선

> 일시: 2026-03-20
> 참석: PO, 위더(Wither)

## 논의 배경

CLAUDE.md가 과도하게 길어진 문제와, Agent/Skill 파일이 PACE 프로젝트에 강하게 결합되어 다른 프로젝트에서 재사용할 수 없는 구조적 문제가 확인되었다. 또한 PO가 프로젝트 전 과정을 위더에게 위임하고 핵심 결정만 체크하고 싶다는 운영 방향이 제시되었다.

## 핵심 결정 사항

### 1. CLAUDE.md 축소 및 재구성

- 263줄 → ~120줄로 축소
- 제품 상세 기능 설명, IA, 팀 상세 테이블 등은 `.docs/planning/`에 위임
- 코드 작업에 필요한 핵심 정보만 유지
- **주요 참조 문서** 섹션 신설 — Agent에 흩어진 문서 경로를 한 곳에 통합
- **Skill/Agent 작동 방식** 섹션 추가 — 관계 명시

### 2. Agent/Skill 범용화

프로젝트 컨텍스트를 Agent/Skill 파일에서 분리하여 CLAUDE.md로 일원화.

- **Agent 7개:** PACE/Spotify 직접 참조 전부 제거. 협업 접점을 이름 기반 → 역할 기반으로 변경. 프로젝트 맥락은 "CLAUDE.md 참조" 한 줄로 대체.
- **Skill 12개:** `.docs/` 경로 통일, PACE_ 파일명 접두사 제거, 도메인 하드코딩 제거.
- **결과:** `.claude/` 폴더를 다른 프로젝트에 복사하고 CLAUDE.md만 새로 쓰면 재사용 가능.

### 3. 에이전트가 프로세스 Skill을 참조하는 구조

에이전트가 작업 중 산출물(명세서, DR, 회의록 등)이 필요할 때 `.claude/skills/` 내 프로세스 Skill 파일을 읽고 동일한 템플릿으로 작성. 이를 통해 누가 만들든 산출물 형식이 일관됨.

각 Agent AGENT.md 끝에 "산출물 작성 시 참조 프로세스" 섹션 추가.

### 4. 에이전트 관리 Skill 추가

- `/add-agent` — 새 에이전트 + 연결 Skill 생성 + CLAUDE.md 팀 테이블 자동 업데이트
- `/remove-agent` — 에이전트 + Skill 제거 + CLAUDE.md 테이블 업데이트
- 역할 기반 참조이므로 팀원 변경 시 다른 Agent 파일 수정 불필요

### 5. PO Inbox 프로세스 (`/inbox`)

위더가 작업 중 PO의 확인·결정·자료 제공이 필요할 때 `.docs/.inbox/`에 요청서를 생성. PO가 나중에 확인하고 처리할 수 있는 비동기 커뮤니케이션 채널.

- 긴급도: 높음(차단)/보통/낮음
- 상태: 대기 → 완료

### 6. 폴더 구조 변경

- `docs/` → `.docs/` — 개발 폴더(apps/, packages/)와 문서 폴더(.docs/, .claude/)를 시각적으로 분리
- `docs/inbox/` → `.docs/.inbox/` — 숨김 폴더로 관리

## 미결 사항

1. **디자인 퀄리티 개선** — PO가 GitHub에서 디자인 관련 Skill을 조사 중. 괜찮은 것을 찾으면 위더가 우리 구조에 통합 예정.
2. **기능 정의 명확화** — `/review`로 현재 명세서 전수 점검 필요.
3. **Swagger API 문서화** — `@nestjs/swagger` 세팅 필요. 이인수에게 위임.

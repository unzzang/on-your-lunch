## 2. Layer 1: 플로우 해부

### 2.1 보내는 사람 플로우 (Selection -> Message -> Payment -> Delivery)

#### Step 1: 진입 (4가지 경로)

| 진입 경로 | 설명 | 마찰 수준 |
|----------|------|----------|
| 친구 탭 생일알림 | 프로필 옆 선물 아이콘 → 생일 특화 홈 | 낮음 (알림 기반 자연스러운 유입) |
| 친구 프로필 | 프로필 페이지 내 선물 버튼 | 낮음 (대상이 이미 특정됨) |
| 채팅창 메뉴 | 대화 중 '+' 메뉴 → 선물하기 | 낮음 (대화 맥락 내 자연스러운 전환) |
| 더보기 탭 | 독립 진입 → 상품 탐색부터 시작 | 중간 (수신자 미특정 상태로 시작) |

출처: [브런치 -- 받는 경험에서 시작하는 카카오톡 선물하기 분석](https://brunch.co.kr/@bydot/6)

4가지 진입점 중 3가지는 수신자가 이미 특정된 상태에서 시작한다. 이는 "누구에게 줄까"라는 의사결정을 생략시켜 전환까지의 인지적 부담을 낮추는 설계이다.

#### Step 2: 상품 탐색 및 선택

| 탐색 방식 | 설명 |
|----------|------|
| 홈 추천 | 개인화 추천, 인기 상품, 시즌 상품 |
| 랭킹 | 급상승/카테고리별/선물테마별 3개 탭 |
| 카테고리 | 메이크업, 향수, 스킨케어 등 세부 분류 |
| 럭스(LuX) | 럭셔리 전문관 (120+ 브랜드, 1만+ 상품) |
| 위시 | 친구가 찜한 상품 확인 |
| 검색 | 키워드 검색 |
| AI 선물탐험 | AI 기반 취향 분석 추천 |

**위시리스트 기능 상세**:
- 받는 사람이 찜한 상품을 보내는 사람이 확인 가능
- 공개/비밀 설정 가능 (공개 시에만 친구에게 노출)
- "친구에게 물어보기" 버튼으로 위시리스트 담기를 요청하는 메시지 발송 가능
- 1:1 채팅방으로 카카오톡 메시지가 자동 발송됨

출처: [카카오 고객센터 -- 위시리스트](https://cs.kakao.com/helps_html/1073195168?locale=ko), [브런치 -- 위시리스트 분석](https://brunch.co.kr/@kakaotalkgift/5)

#### Step 3: 수신자 선택

| 선택 방식 | 설명 |
|----------|------|
| 개인 선물 | 친구 목록에서 1명 선택 |
| 그룹 선물 | 채팅방 또는 친구 목록에서 다수 선택 |

- 받을 친구 삭제 시 확인 버튼이 비활성화되어 홈을 재진입해야 하는 문제

#### Step 6: 선물 전달

- 결제 완료 즉시 수신자에게 카카오톡 메시지로 선물 도착 알림 발송
- 교환권: 바코드/QR 형태로 즉시 사용 가능
- 배송 상품: 수신자가 배송지를 직접 입력한 후 배송 시작

#### 전체 플로우 단계 수

| 유형 | 최소 단계 (프로필 진입 기준) | 최소 단계 (더보기 탭 기준) |
|------|--------------------------|--------------------------|
| 교환권 | 5단계 (프로필 → 상품선택 → 메시지 → 결제 → 완료) | 6단계 (진입 → 상품선택 → 수신자선택 → 메시지 → 결제 → 완료) |
| 배송상품 | 5단계 (동일, 수신자가 별도 배송지 입력) | 6단계 (동일) |

### 2.2 받는 사람 플로우

#### 교환권 수신 플로우

```
카카오톡 메시지 수신
    → 메시지 카드 확인 (사진/영상/음성 포함)
    → "선물 확인하기" 탭
    → 교환권 바코드 표시
    → 매장 방문 → 바코드 제시 → 상품 수령
```

**추가결제**: 교환권 바코드 화면에 "카카오페이로 추가결제" 버튼이 배치되어, 교환권 금액 초과 시 별도 결제 수단 없이 즉시 차액 결제 가능.

출처: [think-note.com -- 선물하기 분석](https://think-note.com/katalkgift/)

#### 배송 상품 수신 플로우

```
카카오톡 메시지 수신
    → 메시지 카드 확인
    → "선물 확인하기" 탭
    → 옵션 선택 (색상/사이즈 등 -- 보내는 사람이 미지정한 경우)
    → 배송지 입력 (이름, 주소, 연락처)
    → "네 배송해주세요" 확인
    → 배송 시작
```

**배송지 입력 경로**: 선물하기 > 선물함 > 미사용 선물 > 상품 선택 > '입력하기' 또는 '옵션 & 배송지 입력'

**배송지 입력 기한**: 일반적으로 7일 이내. 미입력 시 자동 취소 및 환불 처리.

출처: [카카오 고객센터 -- 배송상품](https://ccs.kakao.com/help?service=66&category=276), [eunwooking.com -- 환불 거절 처리법](https://eunwooking.com/entry/kakaotalk-gift-refund-guide)

**옵션 변경 권한 위임**: 보내는 사람에게 "선물 받은 친구가 직접 옵션 변경 가능하니 안심하세요!"라는 마이크로카피로 색상/사이즈 선택 불안을 해소.

출처: [think-note.com -- 선물하기 분석](https://think-note.com/katalkgift/)

## 3. Layer 2: 마이크로카피 수집

### 3.1 선물 거절 관련 마이크로카피

| 상황 | 마이크로카피 (확인된 원문) | 발신 대상 |
|------|--------------------------|----------|
| 거절 알림 (보내는 사람에게) | "선물받으신 분께서 선물에 담긴 소중한 마음만 받으시고 해당 상품은 돌려드리기로 결정하였습니다." | 보내는 사람 |
| 거절 알림 (상세 버전) | "회원님께서 OOO님께 선물하신 것이 받으신 분의 요청에 의해 취소되었습니다. OOO님은 선물에 담긴 소중한 마음만 받으시고, 해당 상품은 돌려드리기로 결정하셨습니다." | 보내는 사람 |

출처: [funissu.com -- 선물 거절 방법](https://funissu.com/kakaotalk-gift-refuse/), [lalawin.com -- 선물 거절](https://lalawin.com/entry/return-katalk-gift)

이 마이크로카피에서 주목할 점은 "거절"이라는 단어를 직접 사용하지 않고, **"소중한 마음만 받으시고"**라는 완충 표현으로 보내는 사람의 감정적 충격을 최소화한다는 것이다. "돌려드리기로 결정하셨습니다"라는 표현도 수동적 거절이 아닌 능동적 선택으로 프레이밍한다.

## 4. Layer 3: 상태별 대응

### 4.1 선물 상태 체계

보내는 사람이 확인할 수 있는 상태는 다음과 같다.

| 상태 | 설명 | 확인 경로 |
|------|------|----------|
| 결제 완료(미사용) | 수신자가 아직 사용하지 않음 | 선물하기 > 선물함 > 주문내역 > 상세보기 |
| 사용완료 | 교환권을 사용함 | 동일 경로 |
| 환불 완료 | 유효기간 경과 후 환불 처리됨 | 동일 경로 |
| 배송 중 | 배송 상품이 출고됨 | 확인 불가 -- 웹 검색 기반 분석의 한계 |
| 거절됨 | 수신자가 선물을 거절함 | 알림톡으로 통보 |

출처: [카카오 고객센터 -- 주문/취소/환불](https://ccs.kakao.com/help?service=66&category=90&device=780)

### 4.2 받는 사람의 선물 상태

| 상태 | 가능한 액션 |
|------|------------|
| 미사용 (교환권) | 사용, 유효기간 연장, 거절, 환불, 옵션 변경 |
| 미사용 (배송상품, 배송지 미입력) | 배송지 입력, 옵션 변경, 거절 |
| 미사용 (배송상품, 배송지 입력 완료) | 배송 대기 |
| 배송 중 | 배송 추적 |
| 배송 완료 | 교환/반품 신청 (7일 이내) |
| 사용완료 | 없음 |
| 만료 | 90% 환불 또는 100% 카카오쇼핑포인트 전환 |

### 4.3 보내는 사람에게의 알림 체계

| 이벤트 | 보내는 사람에게 알림 여부 | 알림 방식 |
|--------|------------------------|----------|
| 수신자 선물 수령 | 확인 불가 -- 웹 검색 기반 분석의 한계 | - |
| 수신자 선물 거절 | O | 알림톡 + 카드사 결제 취소 문자 |
| 수신자 배송지 입력 | 확인 불가 -- 웹 검색 기반 분석의 한계 | - |
| 수신자 교환권 사용 | 보내는 사람이 주문내역에서 상태 확인 가능 | Push 알림 여부는 확인 불가 |
| 수신자 환불 | X (알림 가지 않음) | - |
| 배송지 미입력 자동취소 | X (알림 없음) | - |
| 유효기간 만료 | 확인 불가 -- 웹 검색 기반 분석의 한계 | - |

출처: [a-ha.io -- 선물 받지 않으면 알림](https://www.a-ha.io/questions/4e7d2269dba21671bcfb1bd18d7176c1), [tali.kr -- 환불 시 상대방 알림](https://tali.kr/1863)

주목할 점: **수신자의 환불은 보내는 사람에게 알림이 가지 않는다.** 이는 수신자의 환불 행위가 보내는 사람의 감정을 상하게 하지 않도록 하는 의도적 설계이다. 반면 **거절은 알림이 간다** -- 거절은 명시적 의사 표시이므로 보내는 사람에게 알려야 하지만, 환불은 조용히 처리할 수 있는 옵션을 제공하는 것이다.

## 9. KRQ 응답

### KRQ 1: 선물 선택 -> 결제 -> 전달까지 몇 단계이며 각 단계의 마찰은?

**최소 5단계** (프로필/채팅방 진입 기준): 진입 → 상품선택 → 메시지카드 → 결제 → 완료

**최대 6단계** (더보기 탭 진입 기준): 진입 → 상품탐색 → 수신자선택 → 메시지카드 → 결제 → 완료

| 단계 | 마찰 수준 | 마찰 요인 |
|------|----------|----------|
| 진입 | 낮음 | 4가지 진입점으로 상황별 최적 경로 제공 |
| 상품 탐색/선택 | 중간 | 50만+ 상품 중 선택의 역설. AI/랭킹/위시리스트로 완화 |
| 수신자 선택 | 낮음 (개인) / 중간 (그룹) | 개인은 단순. 그룹은 프로필 미표시 등 UX 문제 |
| 메시지 카드 | 낮음 | 선택적 단계이나 75% 사용. 50여 종 카드 + 미디어 첨부 |
| 결제 | 낮음 | 카카오페이 원터치 결제. 다양한 결제 수단 |

전반적으로 **마찰이 매우 낮은 플로우**로 설계되어 있다. 특히 프로필/채팅방 진입 시 수신자가 사전 확정되어 의사결정 부담이 크게 감소하며, 위시리스트로 상품 선택 부담까지 해소할 수 있다.

### KRQ 2: 받는 사람의 거절/교환/배송지 입력은 어떻게 처리하는가?

**거절**:
- 경로: 더보기 > 받은 선물 > 상품 선택 > "선물 거절" 버튼
- 구매자의 취소 기간(90일) 내 거절 가능
- 즉시 취소 및 환불 처리, 철회 불가
- 보내는 사람에게 정중한 알림톡 발송 ("소중한 마음만 받으시고...")
- 고객센터(1544-2431) 통해서도 신청 가능

**교환**:
- 2024년 3월 1일부터 받는 사람도 직접 교환 신청 가능 (약관 개정)
- 기존에는 보내는 사람만 교환 신청 가능하거나 고객센터 통해 처리
- 단순 변심: 배송 완료 후 7일 이내, 미사용 상태에서만 가능
- 왕복 배송비 고객 부담

**배송지 입력**:
- 보내는 사람이 수신자의 주소를 몰라도 선물 가능
- 수신자가 직접 이름, 주소, 연락처 입력
- "네 배송해주세요" 최종 확인 후 배송 시작
- 입력 기한: 일반적으로 7일 이내
- 미입력 시 자동 취소 및 환불 (보내는 사람에게 알림 없음)
- 옵션(색상/사이즈)도 수신자가 직접 변경 가능

### KRQ 3: 선물 상태(결제/발송/수령)를 보내는 사람에게 어떻게 알리는가?

| 상태 변화 | 알림 방식 |
|----------|----------|
| 결제 완료 | 결제 확인 화면 + 카드사 문자 |
| 수신자 거절 | 알림톡 + 카드사 취소 문자 |
| 수신자 환불 | **알림 없음** (수신자 프라이버시 보호) |
| 배송지 미입력 자동취소 | **알림 없음** |
| 교환권 사용 | 주문내역에서 "사용완료" 상태 확인 (Push 알림 여부 미확인) |
| 유효기간 만료 | 확인 불가 |

**핵심 설계 원칙**: 보내는 사람의 감정을 보호하기 위해, 부정적 이벤트(환불, 미수령 취소)는 알림을 보내지 않는다. 반면 명시적 거절만 알린다. 이는 "관계 보호" 우선의 설계 철학이다.

### KRQ 4: 유효기간/환불 정책은 어떻게 안내하는가?

**유효기간 안내**:
- 기본 유효기간: 3개월 (상품별 상이)
- 만료 전 30일/15일/7일 3단계 알림톡 발송
- 만료 전 연장 신청 시 최대 5년 사용 가능
- 만료 후에는 연장 불가

**환불 정책 안내**:
- 청약철회 기간(94일): 100% 전액 환불 (원 결제 수단)
- 유효기간 만료 후: 90% 환불 (받는 사람에게)
- 2023년 9월~: 카카오쇼핑포인트 100% 전환 옵션 추가
- 2026년 2월 7일~: 100% 환불 가능 (카카오포인트 적립 방식)
- 환불 처리 기간: 영업일 3~5일

## 10. 시사점 정리

### 시사점 1: 감정 중심 프레이밍이 전환율의 핵심

[사실] 카카오톡 선물하기는 "커머스가 아닌 커뮤니케이션"으로 포지셔닝되어 있으며, 메시지 카드 사용률이 75%에 달한다. 채팅창, 프로필, 생일 알림 등 감정적 맥락에서 진입하는 경로가 4가지 중 3가지이다.

[해석] 상품 구매가 아닌 "마음 전달"이라는 프레이밍이 구매 의사결정의 심리적 장벽을 낮춘다. 가격 비교 없이 "이 가격이면 마음을 전할 수 있다"는 판단으로 전환이 일어난다. 최저가 경쟁에서 자유로운 비즈니스 모델이 가능한 이유이다.

[적용] 커머스 서비스에서 단순 상품 나열이 아닌, 사용자의 감정적 동기(감사, 축하, 위로)를 UI 진입점과 메시지 카드에 반영하면 전환율과 객단가를 동시에 높일 수 있다.

### 시사점 2: "받는 사람 경험"이 네트워크 효과의 핵심

[사실] 수신자는 옵션 변경, 배송지 입력, 추가결제, 거절, 교환을 직접 수행할 수 있다. 환불 시 보내는 사람에게 알림이 가지 않으며, 거절 시에도 "소중한 마음만 받으시고"라는 완충 표현을 사용한다.

[해석] 받는 사람의 자율성을 극대화하면서 동시에 보내는 사람의 감정을 보호하는 이중 설계이다. 이는 "선물 받기 → 감동 → 선물 보내기"로 이어지는 네트워크 효과의 핵심 동력이다. 받는 경험이 불편하면 보내는 행위도 줄어들기 때문이다.

[적용] 양면 플랫폼에서 수신자 경험을 1급 시민으로 설계해야 한다. 특히 부정적 상황(거절, 환불)의 마이크로카피는 관계 보호를 최우선으로 해야 하며, 알림 발송 여부도 감정적 영향을 기준으로 판단해야 한다.

### 시사점 3: 주소 모르는 선물의 마찰 제거

[사실] 보내는 사람은 수신자의 주소를 몰라도 선물할 수 있다. 수신자가 직접 배송지와 옵션을 입력하는 구조이다. 보내는 사람에게는 "직접 옵션 변경 가능하니 안심하세요!"라는 카피로 불안을 해소한다.

[해석] 전통적 선물의 최대 마찰("주소를 물어봐야 하는 어색함")을 완전히 제거한 설계이다. 동시에 옵션 선택 책임을 수신자에게 위임함으로써 "색상이 마음에 안 들면 어쩌지"라는 보내는 사람의 불안까지 해결한다.

[적용] 배송형 선물/서비스에서 발신자에게 모든 정보 입력을 요구하지 않고, 수신자에게 자연스럽게 위임하는 플로우를 설계하면 전환 마찰이 크게 줄어든다.

### 시사점 4: 유효기간과 환불의 양면 전략

[사실] 기본 유효기간 3개월, 3단계 만료 알림(30/15/7일), 최대 5년 연장 가능, 만료 후 90% 환불 (2026년부터 100% 포인트 전환). 5년간 10% 수수료로 1,615억 원 수익 추정.

[해석] 짧은 기본 유효기간은 빠른 사용을 유도하고, 연장/환불 옵션은 소비자 보호 불만을 관리한다. 10% 수수료는 "기한 내 사용 유도"라는 행동경제학적 넛지이자 수익원이었으나, 소비자 여론 압박으로 100% 전환(카카오포인트)으로 방향을 전환했다. 이는 "현금이 아닌 포인트"로 돌려줌으로써 플랫폼 내 재소비를 유도하는 전략이다.

[적용] 유효기간을 넛지로 활용하되, 만료 시 페널티보다는 플랫폼 내 포인트 전환으로 재소비를 유도하는 방식이 소비자 반발 없이 LTV를 높이는 전략이 된다.

### 시사점 5: 셀프 선물로 커머스 영역 확장

[사실] 2025년 'For Me' 탭 오픈, 'For Me Week' 전용 프로모션 운영. 셀프 구매 시 "취향템"(다이슨, 프리미엄 디저트)이 인기이며, 타인 선물 시 "실용템"(배달의민족, 스타벅스)이 인기. 영화 카테고리는 전년 대비 5배 성장.

[해석] "선물하기"라는 감성적 프레이밍을 유지하면서 셀프 구매 시장으로 확장하는 전략이다. "나에게 선물"이라는 자기 보상 프레이밍은 일반 쇼핑보다 심리적 허들이 낮다 (죄책감 없는 소비). 기존 선물 인프라(교환권, 배송, 결제)를 재활용하면서 TAM을 크게 확장한다.

[적용] 기존 서비스의 감성적 프레이밍을 유지하면서 사용 시나리오를 확장할 때, 새로운 탭/영역을 분리하되 기존 인프라를 공유하는 방식이 효율적이다.

### 시사점 6: AI 기반 상품 이해와 비교

[사실] 2026년 개편에서 AI가 상품 정보를 분석하여 속성/선물 대상/목적을 태그로 제공하고, 한 줄 요약으로 상품 특성과 추천 대상을 표시한다. 상세 페이지 하단에 유사 상품 3개, 비슷한 가격대 인기 상품 등 추천 영역을 확대했다.

[해석] 50만 개 이상의 상품에서 "선물 맥락에 맞는 상품 발견"이 핵심 과제이며, AI 태깅으로 이를 해결하려는 시도이다. 단순 카테고리 분류를 넘어 "누구에게, 왜, 어떤 상황에" 주는 선물인지를 기준으로 상품을 재분류한다.

[적용] 상품 수가 많은 커머스에서 AI 기반 맥락 태깅(용도, 대상, 상황)은 사용자의 탐색 비용을 크게 줄인다. 특히 선물처럼 "나를 위한 구매"가 아닌 경우, 수신자 관점의 메타데이터가 전환율에 직접 영향을 미친다.

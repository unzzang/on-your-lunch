# 출시 프로세스 (Stage 6)

> 적용 범위: QA 통과 후 프로덕션 배포 및 안정화
> 책임자: PO (출시 결정), 개발팀 (배포)
> 선행 조건: Stage 5 QA Pass 판정

---

## 1. 역할

서비스를 프로덕션에 배포하고, 초기 안정화 기간을 관리한다.

---

## 2. 프로세스

```
Phase 1. 출시 준비
    ↓
Phase 2. 배포
    ↓
Phase 3. 안정화 (출시 후 1~2주)
```

---

### Phase 1. 출시 준비

**절차:**

1. 외부 API 키 발급 + 프로덕션 환경변수 설정
2. 프로덕션 인프라 구성 (백엔드 + 프론트 + DB)
3. 프로덕션 DB 마이그레이션 + 초기 데이터 수집
4. 모니터링/알림 설정

| 활동 | 담당 | 완료 확인 |
|------|------|----------|
| 프로덕션 환경변수 설정 (JWT 시크릿, API 키) | 개발팀 | - [ ] |
| 백엔드 인프라 구성 (Railway 등) | 개발팀 | - [ ] |
| 프론트 인프라 구성 (Vercel 등) | 개발팀 | - [ ] |
| DB 마이그레이션 + 시드 데이터 | 개발팀 | - [ ] |
| 실제 서비스 데이터 수집 | 개발팀 | - [ ] |
| 에러 모니터링 설정 (Sentry 등) | 개발팀 | - [ ] |
| 마케팅 준비 확인 | 한지민 | - [ ] |

**완료 기준:**

- [ ] 프로덕션 환경에서 백엔드 API 정상 응답
- [ ] 프로덕션 환경에서 프론트 페이지 정상 로딩
- [ ] 모니터링 알림 테스트 완료

---

### Phase 2. 배포

**웹(PWA) 배포:**

| 활동 | 담당 | 완료 확인 |
|------|------|----------|
| 백엔드 프로덕션 배포 | 개발팀 | - [ ] |
| 프론트 프로덕션 배포 | 개발팀 | - [ ] |
| DNS/도메인 설정 (커스텀 도메인 있는 경우) | 개발팀 | - [ ] |
| SSL 인증서 확인 | 개발팀 | - [ ] |
| 배포 후 스모크 테스트 | QA팀 | - [ ] |

**앱스토어 배포 (네이티브 앱인 경우):**

| 활동 | 담당 | 완료 확인 |
|------|------|----------|
| 앱스토어 메타데이터 (스크린샷, 설명, 키워드) | PO + 김지수 | - [ ] |
| Apple Developer 등록 + 앱 제출 | PO | - [ ] |
| Google Play 등록 + 앱 제출 | PO | - [ ] |
| 심사 통과 확인 | PO | - [ ] |

**완료 기준:**

- [ ] 프로덕션 URL에서 서비스 접근 가능
- [ ] 핵심 플로우 스모크 테스트 통과

---

### Phase 3. 안정화 (출시 후 1~2주)

| 활동 | 담당 | 주기 |
|------|------|------|
| 서버 모니터링 (에러율, 응답 시간) | 개발팀 | 매일 |
| 에러 리포트 확인 (Sentry 등) | 개발팀 | 매일 |
| 사용자 피드백 수집 | PO + 운영팀 | 매일 |
| 긴급 버그 대응 | 개발팀 + QA팀 | 발생 시 즉시 |
| 핫픽스 배포 (필요 시) | 개발팀 | 발생 시 즉시 |
| 성과 측정 (DAU, 핵심 지표) | 데이터팀 | 주 1회 |

**완료 기준:**

- [ ] 안정화 기간 중 Critical 버그 0건
- [ ] 모니터링 정상 동작 확인
- [ ] 주요 지표 기준치 달성 (PO 판단)

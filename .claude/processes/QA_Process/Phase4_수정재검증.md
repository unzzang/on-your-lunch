# Phase 4. 수정 재검증

> Phase 1~3에서 발견한 이슈를 수정한 뒤 재검증한다.

---

## 절차

1. 발견한 이슈를 심각도(상/중/하)로 분류한다
2. 해당 개발팀에 수정을 요청한다
3. 수정 완료 후 해당 항목만 재검증한다 (전체 재실행 아님)
4. 리그레션 테스트: 수정이 다른 기능에 영향을 주지 않는지 확인한다

## 재검증 체크

- [ ] 모든 상(Critical) 이슈 수정 + 재검증 통과
- [ ] 모든 중(Major) 이슈 수정 + 재검증 통과
- [ ] 리그레션 테스트 통과 (수정 전 정상이던 기능이 여전히 정상인지)
- [ ] 수정 후 빌드/린트 재확인

## 최종 판정

모든 재검증이 통과하면 QA 체크리스트를 완성하고 최종 판정을 내린다.

**QA 체크리스트:** `kits/templates/documents/QA_Checklist_Template.md`를 복사하여 `docs/004_planning/qa/`에 저장.

**버그 리포트:** `docs/004_planning/qa/`에 기록.

```
판정자: 진도준
판정: Pass / Conditional Pass / Fail
판정일: YYYY-MM-DD
```

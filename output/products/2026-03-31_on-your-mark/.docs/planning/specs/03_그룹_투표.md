# 03. 그룹 투표

> 작성일: 2026-03-30
> 작성자: 도리
> 상태: 초안

## 개요

동료들과 점심 메뉴를 정할 때 카톡에서 "뭐 먹을까?" → "아무거나" → "아무거나 말고"가 반복되며 10분 이상 소모된다. AI가 좁혀준 메뉴 후보 4개로 투표방을 만들고, 동료들이 5~10분 내에 투표로 확정하는 기능이다.

## 사용자 스토리

- 점심 메뉴를 정하려는 신입 사원으로서, AI가 추천한 메뉴로 투표방을 한 번에 생성하고 싶다. 매번 후보를 직접 나열하지 않기 위해.
- 투표를 만든 사원으로서, 카카오톡으로 동료에게 초대 링크를 보내고 싶다. 별도 설명 없이 링크만 누르면 참여할 수 있게 하기 위해.
- 초대받은 동료로서, 앱을 설치하지 않아도 웹에서 바로 투표하고 싶다. 앱 설치 과정 없이 30초 안에 투표를 끝내기 위해.
- 투표를 만든 사원으로서, 마감 시간이 되면 자동으로 결과가 확정되길 원한다. 마감 타이밍을 직접 신경 쓰지 않기 위해.

## 기능 상세

### 3-1. 투표방 생성

**동작:**

1. 사용자가 AI 메뉴 추천 화면(02_AI_메뉴_추천)에서 "이걸로 투표하기" 버튼을 탭한다.
2. 현재 표시된 AI 추천 메뉴 4개가 투표 후보로 자동 설정된다.
3. 투표방 설정 화면으로 이동한다. 설정 항목:
   - 투표방 제목: 기본값 "오늘의 점심 투표" (수정 가능, 최대 30자)
   - 마감 방식: 타이머(5분/10분 중 택1) 또는 수동 마감 — 기본값 5분
   - 투표 후보 목록: AI 추천 메뉴가 미리 채워짐 (삭제만 가능, 최소 2개 유지)
4. "투표 시작" 버튼 탭 → 투표방 생성 완료 → 투표방 화면으로 이동
5. 투표방 생성과 동시에 타이머가 시작된다 (타이머 마감 선택 시).

**정책:**

| 항목 | 규칙 |
|------|------|
| 투표 후보 수 | 최소 2개, 최대 4개 |
| 동시 활성 투표방 | 사용자당 1개. 기존 투표방이 진행 중이면 새로 생성 불가 |
| 투표방 유효시간 | 생성 후 최대 1시간. 마감하지 않아도 1시간 후 자동 만료 |
| 투표방 생성 권한 | 앱 설치 + 로그인한 사용자만 가능 |
| 1명 투표 유효 | 참여자가 생성자 1명뿐이어도 투표 및 마감이 유효하다. 혼자 투표한 메뉴가 확정된다 |
| 마감 타이머 옵션 | 5분, 10분, 수동 마감 (3가지) |

**에러 케이스:**

| 상황 | 대응 |
|------|------|
| AI 추천 메뉴가 아직 로드되지 않은 상태에서 "이걸로 투표하기" 탭 | 버튼 비활성화. 추천 메뉴 로드 완료 후 활성화 |
| 이미 진행 중인 투표방이 있는 상태에서 새 투표방 생성 시도 | "진행 중인 투표가 있어요. 마감 후 새로 만들 수 있어요." 토스트 + 기존 투표방으로 이동 버튼 |
| 후보 메뉴를 삭제하여 1개만 남은 경우 | "투표 시작" 버튼 비활성화. "후보가 2개 이상 필요해요" 안내 문구 표시 |
| 네트워크 오류로 투표방 생성 실패 | "네트워크 연결을 확인해주세요" 토스트 + 재시도 버튼 |

### 3-2. 초대 링크 공유

**동작:**

1. 투표방 생성 완료 후, 투표방 화면 상단에 "동료 초대" 영역이 표시된다.
2. 공유 방식 2가지:
   - **카카오톡 공유:** 카카오톡 공유 API를 통해 메시지 전송. 메시지에 투표 제목, 후보 메뉴 목록, 남은 시간, "투표하기" 버튼 포함.
   - **링크 복사:** 투표방 URL을 클립보드에 복사. "링크가 복사되었어요" 토스트 표시. 사용자가 원하는 메신저에 직접 붙여넣기.
3. 초대 링크 형식: `https://onyourmark.app/vote/{투표방ID}` (8자리 영숫자 랜덤 ID)

**웹 참여 플로우 (DR-003 결정사항):**

앱 미설치자가 초대 링크를 탭했을 때:

1. 모바일 웹 브라우저에서 투표 참여 페이지가 열린다.
2. 닉네임 입력란 표시 — 최소 1자, 최대 10자. 회원가입/로그인 불필요.
3. 닉네임 입력 후 "참여하기" 탭 → 투표 화면 진입.
4. 투표 완료 후 결과 화면 하단에 "앱에서 더 편하게 쓰기" 앱 설치 유도 배너 표시 (필수 아님, 닫기 가능).

앱 설치자가 초대 링크를 탭했을 때:

1. 딥링크로 앱이 열리며 해당 투표방으로 바로 이동.
2. 이미 로그인 상태이므로 닉네임 입력 불필요. 앱 프로필의 닉네임이 자동 사용.

**정책:**

| 항목 | 규칙 |
|------|------|
| 초대 링크 유효기간 | 투표방과 동일 (마감 또는 최대 1시간) |
| 링크 접근 제한 | 링크를 가진 누구나 접근 가능 (별도 인증 없음) |
| 웹 참여자 닉네임 중복 | 같은 투표방 내 닉네임 중복 허용하지 않음 |
| 웹 참여자 기능 제한 | 투표 참여만 가능. 투표방 생성, 수동 마감은 불가 |
| 카카오톡 공유 메시지 | 제목: "{닉네임}님이 점심 투표에 초대했어요!", 본문: 후보 메뉴 목록 나열, 버튼: "투표하기" |

**에러 케이스:**

| 상황 | 대응 |
|------|------|
| 마감된 투표방 링크에 접근 | "이미 마감된 투표예요" 안내 + 결과 화면 표시 (읽기 전용) |
| 만료(1시간 초과)된 투표방 링크에 접근 | "만료된 투표 링크예요" 안내 |
| 웹에서 닉네임 입력 시 투표방 내 중복 닉네임 | "이미 사용 중인 이름이에요. 다른 이름을 입력해주세요." |
| 카카오톡 미설치 상태에서 카카오톡 공유 시도 | 카카오톡 공유 버튼 숨김. 링크 복사만 표시 |
| 닉네임 미입력 또는 공백만 입력 | "참여하기" 버튼 비활성화 |

### 3-3. 실시간 투표

**동작:**

1. 투표방에 입장하면 후보 메뉴 카드 목록이 표시된다. 각 카드에는 메뉴명과 현재 득표수가 표시된다.
2. 투표 방식: **단일 선택** — 후보 중 1개만 선택 가능.
3. 메뉴 카드를 탭하면 즉시 투표가 반영된다. 선택한 카드에 체크 표시 + 본인 닉네임 태그가 붙는다.
4. **투표 변경 가능** — 마감 전까지 다른 메뉴를 탭하면 기존 선택이 해제되고 새로운 메뉴로 변경된다.
5. 실시간 결과 표시:
   - 각 메뉴 카드에 현재 득표수 숫자 표시
   - 각 메뉴 카드 하단에 투표한 사람의 닉네임 나열 (최대 5명까지 표시, 초과 시 "+N명" 표기)
   - 최다 득표 메뉴 카드에 "1등" 뱃지 표시 (동점 시 해당 메뉴 모두에 "공동 1등" 뱃지)
6. 투표방 상단에 참여자 수와 남은 시간(타이머 마감인 경우) 표시.
7. 새로운 참여자가 입장하면 참여자 수가 실시간 반영된다.

**정책:**

| 항목 | 규칙 |
|------|------|
| 투표 방식 | 단일 선택 (1인 1표) |
| 투표 변경 | 마감 전까지 무제한 변경 가능 |
| 최대 참여자 수 | 20명. 초과 시 입장 불가 |
| 투표방 생성자의 투표 | 생성자도 투표에 참여해야 함 (자동 투표 아님) |
| 결과 공개 방식 | 실시간 공개 (투표 즉시 결과 반영, 비밀 투표 아님) |
| 미투표 참여자 | 입장만 하고 투표하지 않은 참여자는 결과에 미반영. 참여자 목록에는 표시 |

**에러 케이스:**

| 상황 | 대응 |
|------|------|
| 참여자 수 20명 초과 시 입장 시도 | "참여 인원이 가득 찼어요 (20/20)" 안내 |
| 마감된 투표방에서 투표 시도 | 투표 버튼 비활성화. "투표가 마감되었어요" 안내 |
| 네트워크 끊김 상태에서 투표 시도 | "네트워크 연결을 확인해주세요" 토스트. 재연결 시 최신 상태 동기화 |
| 투표방 진입 후 브라우저/앱을 닫았다가 재진입 | 기존 투표 상태 유지. 마감 전이면 투표 변경 가능 |

### 3-4. 투표 마감

**동작:**

1. 마감 조건 (다음 중 하나 충족 시 마감):
   - **타이머 만료:** 설정된 시간(5분 또는 10분)이 경과하면 자동 마감.
   - **수동 마감:** 투표방 생성자가 "지금 마감" 버튼을 탭하면 즉시 마감. (수동 마감 모드뿐 아니라 타이머 모드에서도 조기 마감 가능)
   - **전원 투표 완료:** 투표방 참여자 전원이 투표를 완료하면 자동 마감 (참여자 수 2명 이상일 때).
2. 마감 시 결과 화면:
   - 1등 메뉴가 화면 중앙에 강조 표시 ("오늘의 점심은 {메뉴명}!")
   - 전체 메뉴별 득표수 + 득표율(%) 표시
   - 각 메뉴에 투표한 사람 닉네임 전체 표시
   - "배달 주문하기" 버튼 → 앱 사용자는 배달앱 딥링크 화면(04_배달_연동_및_기록)으로 이동. 웹 참여자는 배달앱 모바일 웹 URL로 이동 (배달의민족: `m.baemin.com/search?query={메뉴명}`, 쿠팡이츠: `m.coupangeats.com/search?keyword={메뉴명}`)
   - "결과 공유하기" 버튼 → 결과 이미지 또는 텍스트를 카카오톡/링크로 공유
3. 마감 결과는 투표방 유효시간(최대 1시간) 동안 조회 가능.

**동점 처리:**

| 조건 | 처리 |
|------|------|
| 2개 이상 메뉴가 동점 1위 | 동점 메뉴 중 **먼저 최다 득표에 도달한 메뉴**를 1등으로 확정. (예: A와 B가 3표 동점이지만 A가 3표에 먼저 도달 → A 확정) |
| 전원 미투표 상태에서 타이머 만료 | "아무도 투표하지 않았어요" 안내. 1등 메뉴 없이 결과 화면 표시. "다시 투표하기" 버튼 제공 |
| 1명만 투표한 상태에서 마감 | 해당 메뉴를 1등으로 확정 |

**정책:**

| 항목 | 규칙 |
|------|------|
| 마감 취소 | 불가. 마감은 되돌릴 수 없음 |
| 수동 마감 권한 | 투표방 생성자만 가능. 생성자가 탈퇴하거나 앱을 종료한 경우, 수동 마감은 불가하며 타이머 마감만 유효하다 |
| 마감 후 투표 변경 | 불가 |
| 마감 알림 | 마감 시 투표방 참여자 전원에게 푸시 알림 발송 (앱 사용자 한정). 웹 참여자에게는 화면 내 알림 표시 |
| 타이머 30초 전 경고 | 마감 30초 전 투표방 내 "곧 마감됩니다" 배너 표시 |
| 결과 자동 기록 | 확정된 메뉴는 "오늘의 점심 기록"(04_배달_연동_및_기록)에 자동 저장 |

**에러 케이스:**

| 상황 | 대응 |
|------|------|
| 수동 마감 버튼 탭 시 네트워크 오류 | "네트워크 연결을 확인해주세요" 토스트 + 재시도 버튼 |
| 타이머 만료 시점에 서버 통신 지연 | 클라이언트에서 타이머 종료 UI를 먼저 표시하고, 서버 응답 수신 후 최종 결과로 갱신 |
| 투표방 생성자가 앱을 종료한 상태에서 타이머 만료 | 서버에서 자동 마감 처리. 생성자 재진입 시 결과 화면 표시 |
| "다시 투표하기" 선택 시 | 기존 투표방은 만료 처리. 동일 메뉴 후보로 새 투표방 생성 (동시 활성 투표방 1개 정책 유지) |

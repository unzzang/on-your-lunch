# 홈 화면 설계

## 1. 화면 목적

앱의 첫 화면. 오늘의 점심 추천 3곳을 카드 형태로 보여주고, 필터로 조건을 조절할 수 있다. 사용자가 앱을 열었을 때 "오늘 뭐 먹지?"에 대한 답을 바로 볼 수 있어야 한다.

## 2. 화면 구조

```
┌──────────────────────────────┐
│ 상단 바                        │
│  [온유어런치 로고]     [알림🔔] │

├──────────────────────────────┤
│ 필터 영역                      │
│  [한식][중식][일식][양식]...    │
│  [도보 5분|●10분|15분]         │

│                               │

│                               │
```

## 3. 영역별 상세

### 3.1. 상단 바

| 요소 | 위치 | 동작 |
|------|------|------|
| 온유어런치 로고 | 좌측 | 탭 시 홈 최상단으로 스크롤 |
| 알림 아이콘 | 우측 | 탭 시 알림 목록 (이후 구현, MVP에서는 비활성) |

### 3.2. 인사 영역

- "{닉네임}님, 오늘 점심 뭐 먹을까요?" 고정 텍스트
- 스크롤 시 상단 바에 가려짐

### 3.3. 필터 영역

- 카테고리: 가로 스크롤 칩. 탭하면 활성화(색상 채워짐), 재탭하면 해제. 복수 선택 가능.
- 도보 거리: 3개 탭 (5분/10분/15분). 단일 선택. 기본값 10분.
- 가격대: 3개 탭 (1만원 이하/1~2만원/2만원 이상). 단일 선택. 기본값 없음(전체).
- 필터 변경 시 추천 카드 즉시 갱신 (새로고침 횟수 차감 안 함)
- 스크롤 시 상단에 고정(sticky)

### 3.4. 추천 카드

- 세로로 3장 나열 (스크롤)
- 각 카드 구성:
  - 대표 사진 (가로 풀 너비, 높이 고정. 사진 없으면 카테고리별 기본 일러스트)
  - 식당명 (볼드)
  - 카테고리 · 도보 시간 · 가격대 (한 줄)
  - 즐겨찾기 하트 아이콘 (카드 우상단)
  - 이전에 방문한 식당이면: "★ 4.0 · 3번 방문" 뱃지 표시
- 카드 탭 → 식당 상세 화면으로 이동

### 3.5. 새로고침 버튼

- 추천 카드 3장 아래에 위치
- "다른 추천 보기 (남은 횟수/5)" 텍스트
- 탭 → 3장 새로 갱신 (애니메이션: 카드 페이드 아웃 → 새 카드 페이드 인)
- 5회 소진 시 → 비활성화 + "오늘 추천을 모두 사용했어요. 탐색에서 직접 찾아보세요!" + 탐색 탭 이동 링크

### 3.6. 하단 탭 바

| 탭 | 아이콘 | 이동 화면 |
|----|--------|-----------|
| 홈 | 집 아이콘 | 현재 화면 |
| 탐색 | 돋보기 아이콘 | 식당 탐색 (지도/리스트) |
| 이력 | 캘린더 아이콘 | 먹은 이력 캘린더 |
| 마이 | 사람 아이콘 | 마이페이지 |

- 현재 활성 탭: 아이콘 + 텍스트 색상 강조
- 항상 화면 하단에 고정

## 4. 화면 전환 (네비게이션)

| 출발 | 동작 | 도착 |
|------|------|------|
| 홈 → 추천 카드 탭 | 탭 | 식당 상세 화면 |
| 홈 → 필터 변경 | 탭 | 홈 (추천 갱신) |
| 홈 → 새로고침 | 탭 | 홈 (새 추천 3장) |
| 홈 → 하단 탭 | 탭 | 각 탭 화면 |
| 홈 → 즐겨찾기 하트 | 탭 | 홈 (하트 토글, 화면 유지) |

## 5. 상태별 화면

| 상태 | 화면 표시 |
|------|-----------|
| 정상 | 추천 카드 3장 + 필터 + 새로고침 버튼 |
| 추천 후보 없음 | "조건에 맞는 식당을 찾지 못했어요" + 필터 초기화 버튼 |
| 새로고침 소진 | 새로고침 버튼 비활성 + 탐색 안내 |
| 네트워크 오류 | "인터넷 연결을 확인해주세요" + 재시도 버튼 |
| 첫 사용 (이력 0건) | 정상 표시 (이력 없이도 취향 기반 추천) |

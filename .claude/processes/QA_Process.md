# QA 프로세스 (Stage 5)

> 적용 범위: 개발 완료된 기능의 품질 검증
> 책임자: 진도준 (QA 최종 판정권)
> 선행 조건: Stage 4 Phase 5 Definition of Done 충족

---

## 1. 역할

개발 산출물이 기능 명세서와 일치하는지, 안정적으로 동작하는지 검증한다. 진도준이 Pass/Fail 최종 판정권을 갖는다.

## 2. 프로세스

```
Phase 1 → Phase 2 → Phase 3 → Phase 4
```

### Phase 1. 정적 검증

- 빌드 성공 여부
- 테스트 통과 여부
- 린트 에러 0건
- 타입 체크 통과

### Phase 2. 통합 검증

- API 계약 일치 (`shared-types/` 기준)
- 환경 설정 정합성
- 프론트-백 연동 동작

### Phase 3. 기능 검증

- 스모크 테스트 (핵심 플로우 정상 동작)
- 기능 명세서 기준 상세 테스트
- 에러 시나리오 확인 (네트워크 에러, 빈 상태, 권한 등)

### Phase 4. 수정 재검증

- 버그 리포트 → 개발팀 수정 → 재검증
- 리그레션 테스트 (수정이 다른 기능에 영향 없는지)
- Critical/Major 미해결 0건 확인

**QA 체크리스트:** `QA_Checklist_Template.md`를 복사하여 사이클마다 사용.

**버그 리포트 위치:** `.docs/004_planning/specs/` 하위 또는 별도 이슈 트래커.

## 3. 완료 기준

- [ ] Phase 1~3 전체 통과
- [ ] Critical/Major 버그 0건
- [ ] 진도준 최종 판정: Pass
- [ ] QA 체크리스트 완료본 저장

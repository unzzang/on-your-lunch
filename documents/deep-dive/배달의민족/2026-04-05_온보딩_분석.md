## 1. KRQ 요약 답변

### KRQ 1: 첫 실행~첫 주문까지 몇 단계이며 각 단계에서 무엇을 요구하는가?

첫 실행부터 첫 주문까지 총 4개 페이즈, 약 15~20개 개별 스텝으로 구성된다.

| 페이즈 | 스텝 수 | 요구 사항 |
|--------|---------|-----------|
| A. 초기 설정 (첫 실행) | 4~5 스텝 | 접근 권한 허용, 위치 기반 약관 동의, 마케팅 수신 선택, 주소 등록 |
| B. 회원가입 (선택적) | 5~6 스텝 | 소셜 로그인 또는 휴대폰 인증, 본인 확인, 약관 동의, 이메일 입력 |
| C. 탐색 및 메뉴 선택 | 3~5 스텝 | 배달 방식 선택, 카테고리/검색, 가게 선택, 메뉴 선택, 장바구니 담기 |
| D. 결제 및 주문 | 3~4 스텝 | 결제수단 등록(배민페이), 주소 확인, 주문하기 |

배민의 핵심 설계 원칙은 **"회원가입 없이도 탐색 가능"**이다. 회원가입은 주문 시점 또는 my배민 탭 진입 시에만 요구되며, 첫 실행 직후 주소만 등록하면 홈 화면에서 가게 탐색이 가능하다.

출처: [funissu.com/baemin-user-signup](https://funissu.com/baemin-user-signup/), [easybaemin.com](https://easybaemin.com/basic/5/)

### KRQ 2: 권한(위치/알림/마케팅)을 언제, 어떤 문구로 요청하는가?

배민의 권한 요청은 3단계로 분리되어 있다.

| 권한 | 요청 시점 | 요청 방식 | 필수/선택 |
|------|-----------|-----------|-----------|
| 위치 기반 서비스 약관 | 첫 실행 직후 (스텝 2~3) | 자체 약관 동의 화면 + OS 권한 팝업 | 약관 동의는 필수, OS 위치 권한은 선택 |
| 알림(푸시) | 첫 실행 초기 단계 | OS 시스템 팝업 | 선택 |
| 마케팅/이벤트 수신 | 주소 등록 전후 | 자체 화면 ("나중에 받기" 옵션 제공) | 선택 |
| 기타 (카메라, 주소록, 생체인증 등) | 해당 기능 사용 시점 | OS 시스템 팝업 | 선택 |

위치 권한의 경우, 사용자가 "현재 위치로 찾기" 버튼을 누를 때 OS 권한 팝업이 뜨는 구조이며, 거부해도 직접 주소 입력으로 대체 가능하다.

출처: [maily.so/tipster/posts/aa72ae](https://maily.so/tipster/posts/aa72ae), [easybaemin.com/basic/2](https://easybaemin.com/basic/2/)

### KRQ 3: 이탈 방지 장치(스킵, 나중에, 혜택)는 무엇인가?

배민은 다음 5가지 이탈 방지 장치를 운영한다.

| 장치 | 설명 | 적용 시점 |
|------|------|-----------|
| 회원가입 지연 | 가입 없이 탐색 가능. 주문 시점에만 가입 요구 | 첫 실행 ~ 홈 화면 |
| "나중에 받기" 옵션 | 마케팅 수신 동의를 강제하지 않음 | 마케팅 알림 화면 |
| 위치 권한 대체 | 위치 권한 거부 시 수동 주소 입력으로 대체 | 주소 설정 화면 |
| 첫주문 쿠폰팩 | 신규 가입 시 최대 14,000원 쿠폰팩 자동 지급 (배민B마트 10,000원 + CU 4,000원) | 가입 직후 |
| 친구초대 코드 | 초대 코드 입력 시 10,000원 할인 쿠폰 (15,000원 이상 주문 시 사용 가능) | my배민 > 이벤트 |

출처: [funissu.com/baemin-user-signup](https://funissu.com/baemin-user-signup/), [easybaemin.com/basic/2](https://easybaemin.com/basic/2/)

### KRQ 4: 개인화를 위해 어떤 정보를 어느 시점에 수집하는가?

| 수집 정보 | 수집 시점 | 개인화 활용 |
|-----------|-----------|-------------|
| 배달 주소 (위치) | 첫 실행 즉시 | 주변 가게 목록, 배달 가능 여부 판단 |
| 주문 이력 | 주문 완료 이후 축적 | 시간대별 맞춤 추천, "이전 주문" 리스트 |
| 검색 키워드 | 검색 시 | 검색 추천어, 연관 가게 |
| 찜한 가게 | 찜 버튼 클릭 시 | 찜 목록 기반 추천 |
| 리뷰 작성 데이터 | 리뷰 작성 시 | 취향 프로필 구축 (추정) |
| 앱 사용 패턴 | 상시 | 개인별 맞춤형 홈 화면 노출 순서 |

배민은 2024년 홈 화면 개편을 통해 "고객 개인별 맞춤형으로 노출 화면을 제공"하는 방식을 도입했다. 사용자의 주문 패턴과 탐색 행동을 기반으로 홈 화면 내 가게 순서와 추천 콘텐츠가 달라진다.

출처: [news1.kr](https://www.news1.kr/industry/sb-founded/5377241), [asiae.co.kr](https://www.asiae.co.kr/article/2024040809452814560)

## 2. Layer 1: 플로우 해부

### 2.1 전체 플로우 다이어그램

```
[앱 설치] → [첫 실행]
    ↓
[스플래시 화면] (배민 로고, 약 1~2초)
    ↓
[접근 권한 안내 팝업] — 선택 권한 목록 표시
    ↓
[위치 기반 서비스 약관 동의] — 필수 체크 → "시작하기"
    ↓
[마케팅 알림 수신 선택] — "나중에 받기" 가능
    ↓
[알림 권한 요청] — OS 시스템 팝업 (iOS/Android)
    ↓
[주소 설정 화면]
    ├── "현재 위치로 찾기" → OS 위치 권한 팝업 → 자동 주소 설정
    └── 직접 검색 → 도로명 주소 입력 → 상세 주소 입력 → "주소등록"
    ↓
[홈 화면 진입] ← ★ 회원가입 없이 도달
    ↓
[가게 탐색] — 카테고리/검색/배너/추천
    ↓
[가게 상세] → [메뉴 선택] → [장바구니]
    ↓
[주문하기] → 회원가입 필요 시점
    ├── [소셜 로그인] (카카오/네이버/애플)
    └── [휴대폰번호 가입] → 본인인증 → 약관동의 → 이메일 입력
    ↓
[결제수단 등록] — 배민페이 카드 등록 (카메라 스캔 또는 수동 입력)
    ↓
[주문 확인] → "주문하기" 버튼
    ↓
[주문 완료] → 실시간 배달 추적
```

### 2.2 페이즈별 상세

#### 페이즈 A: 초기 설정 (첫 실행)

**스텝 A-1: 스플래시 화면**
- 배민 로고 표시
- 소요 시간: 약 1~2초 (확인 불가 -- 웹 검색 기반 분석의 한계)
- 사용자 입력: 없음

**스텝 A-2: 접근 권한 안내**
- 선택 권한 목록을 한 화면에 표시
- 내용: 저장 공간, 위치, 카메라, 주소록, 생체 인증, 마이크/스피커/블루투스
- 각 권한별 사용 목적 설명 포함
- "허용에 동의하지 않으셔도 배달의민족을 이용할 수 있습니다" 안내

출처: [apps.apple.com 배달의민족](https://apps.apple.com/kr/app/%EB%B0%B0%EB%8B%AC%EC%9D%98%EB%AF%BC%EC%A1%B1-%EB%B0%B0%EB%8B%AC%ED%8C%81-%EB%AC%B4%EB%A3%8C-%EB%B0%B0%EB%AF%BC%ED%81%B4%EB%9F%BD/id378084485)

**스텝 A-3: 위치 기반 서비스 약관 동의**
- 필수 항목: 위치 기반 서비스 약관 동의 체크박스
- CTA: "시작하기" 버튼
- 이 단계를 거치지 않으면 앱 진입 불가

출처: [funissu.com/baemin-user-signup](https://funissu.com/baemin-user-signup/)

**스텝 A-4: 마케팅 알림 수신 선택**
- 이벤트 및 혜택 정보 수신 동의
- 선택사항으로, "나중에 받기" 옵션 제공
- 별도 화면으로 분리되어 약관 동의와 혼동되지 않도록 설계

출처: [easybaemin.com/basic/2](https://easybaemin.com/basic/2/)

**스텝 A-5: 주소 설정**
- 두 가지 경로:
  1. "현재 위치로 찾기" → OS 위치 권한 팝업 → GPS 기반 자동 주소 설정
  2. 직접 검색 → 지번/도로명/건물명 입력 → 검색 결과 선택 → 상세 주소 입력 → "주소등록"
- 위치 권한 거부 시에도 직접 입력으로 진행 가능

출처: [easybaemin.com/basic/2](https://easybaemin.com/basic/2/)

#### 페이즈 B: 회원가입 (주문 시점에 요구)

**스텝 B-1: 로그인 화면 진입**
- my배민 탭 → "로그인하고 시작하기" 버튼, 또는 주문하기 시점에 자동 전환
- 로그인 옵션: 카카오, 네이버, 페이스북, 애플, 휴대폰번호

출처: [funissu.com/baemin-user-signup](https://funissu.com/baemin-user-signup/)

**스텝 B-2: 소셜 로그인 경로 (카카오/네이버/애플)**
- 해당 플랫폼의 인증 화면으로 전환
- 기존 계정 연동으로 별도 정보 입력 최소화
- 소요 시간: 약 10~30초 (추정)

**스텝 B-3: 휴대폰번호 가입 경로**
1. "휴대폰번호로 계속하기" 선택
2. 이름, 주민등록번호 앞 7자리, 휴대폰번호 입력
3. 통신사 선택
4. "본인 인증하기" → "동의하고 인증번호 받기"
5. SMS 6자리 인증번호 입력
6. 필수/선택 동의사항 체크 → "다음"
7. 이메일 주소 입력 → "확인" → 가입 완료

출처: [easybaemin.com/basic/5](https://easybaemin.com/basic/5/)

#### 페이즈 C: 탐색 및 메뉴 선택

**홈 화면 구조 (2024년 개편 이후)**

상단 탭 구조:
- 배민배달 | 가게배달 | 장보기/쇼핑 | 배민선물하기

각 탭 선택 시 해당 서비스의 카테고리와 가게 목록이 표시된다.

하단 네비게이션:
- 홈 | 찜 | 주문내역 | 배민스토어(확인 불가) | my배민

배달 방식 선택:
- 알뜰배달, 한집배달, 가게배달, 픽업 중 선택

출처: [news1.kr](https://www.news1.kr/industry/sb-founded/5377241), [easybaemin.com/basic/3](https://easybaemin.com/basic/3/)

#### 페이즈 D: 결제 및 주문

**결제수단 등록 (배민페이)**
1. 마이배민 → "배민페이 간편하게 등록하기"
2. "결제수단 추가" (+ 아이콘)
3. "카드 등록하기"
4. 카메라로 카드 스캔 (또는 수동 입력)
5. 카드 정보 확인 → "완료"
6. 보안코드(CVC) + 카드 비밀번호 입력 → "동의하고 완료"
7. 결제용 비밀번호 6자리 설정

결제수단 옵션:
- 배민페이머니, 신용/체크카드, 토스페이
- 기타: 애플페이, 네이버페이, 카카오페이
- 만나서 결제 (현금/카드)

출처: [easybaemin.com/basic/6](https://easybaemin.com/basic/6/)

### 2.3 전환 퍼널 분석

```
앱 설치        ─── 100%

회원가입 완료   ─── (이탈 포인트: 본인인증 번거로움)
  ↓
결제수단 등록   ─── (이탈 포인트: 카드 정보 입력)
  ↓
첫 주문 완료    ─── 전환 성공
```

## 3. Layer 2: 마이크로카피 수집

### 3.1 온보딩 단계 마이크로카피

아래 표는 웹에서 수집 가능한 배민 온보딩 관련 마이크로카피를 정리한 것이다. 실제 앱 화면에서의 정확한 문구와 차이가 있을 수 있다.

| 화면 | 요소 | 수집된 문구 | 톤 | 의도 |

### 3.4 UX 라이팅 원칙 (배민 공식)

배민은 전사적 UX 라이팅 TF를 운영하며, 다음 원칙을 따른다:

1. **사용자 관점 중심**: 서비스 맥락을 파악하여 사용자에게 더 중요한 정보를 우선 표시 (예: "함께주문"에서 "선택을 완료한 사람 수" → "남은 사람 정보"로 변경)
2. **균형잡힌 커뮤니케이션**: 사용자 편의성과 서비스 필요성 사이의 균형
3. **기대치 설정**: 과정이 얼마나 걸리는지 안내 (예: "2분 만에 끝내기" - 직접 측정한 시간)
4. **핵심 전달**: 필요한 이유를 핵심만 전달하여 신뢰 구축

배민은 UX 라이터(유다정)가 디자인옵스팀에서 시작하여 UX 라이팅 TF로 발전시켰으며, 디자인 피드백 단계에 라이팅 검토를 통합하는 프로세스를 운영한다.

## 9. 시사점 정리

### 시사점 1: 회원가입 지연 전략

- **[사실]** 배민은 첫 실행 시 회원가입을 요구하지 않는다. 주소 설정만으로 홈 화면 진입과 가게 탐색이 가능하며, 회원가입은 주문 시점에만 요구한다.
- **[해석]** "가입 전 가치 체험" 패턴이다. 사용자가 서비스의 핵심 가치(주변 가게 목록, 메뉴, 리뷰)를 확인한 후에 가입을 결정하게 하여, 가입 전환율과 첫 주문 전환율을 동시에 높인다. 온보딩 초기 이탈(권한 요청 → 회원가입으로 이어지는 피로)을 방지한다.
- **[적용]** 온유어런치에서도 "주소 설정 → 홈 화면 진입 → 가게 탐색" 플로우를 회원가입 없이 허용하는 것을 검토할 수 있다. 단, 온유어런치의 핵심 기능(점심 추천)이 개인화에 의존한다면, 최소한의 정보(위치+선호도)만 먼저 수집하고 가입은 뒤로 미루는 하이브리드 접근이 가능하다.

### 시사점 2: 권한 요청의 분리와 대체 수단

- **[사실]** 배민은 위치 기반 약관 동의(필수)와 OS 위치 권한(선택)을 분리하고, 권한 거부 시 직접 주소 입력으로 대체 가능하게 한다. 마케팅 수신도 별도 화면에서 "나중에 받기" 옵션과 함께 제공한다.
- **[해석]** 각 권한을 요청하는 시점과 맥락을 분리함으로써, 사용자가 "모 아니면 도"로 판단하지 않게 한다. 권한 하나를 거부해도 서비스 이용에 지장이 없다는 안내("허용에 동의하지 않으셔도 배달의민족을 이용할 수 있습니다")가 거부감을 줄인다.
- **[적용]** 온유어런치에서 위치 권한 요청 시, (1) 필수 약관 동의 → (2) 기능 사용 시점에 OS 권한 요청 → (3) 거부 시 수동 입력 대체의 3단계 패턴을 적용할 수 있다. 알림 권한도 첫 주문 완료 후 "배달 상태를 실시간으로 받아보시겠어요?" 시점에 요청하는 것이 전환율이 높을 것으로 추론된다.

### 시사점 3: 소셜 로그인 우선 + 본인인증 보완

- **[사실]** 배민은 카카오/네이버/애플/페이스북 소셜 로그인과 휴대폰번호 가입을 제공한다. 기존 이메일 가입은 폐지했다. 소셜 로그인을 선택하면 별도 본인인증 없이 가입이 가능하고, 휴대폰번호 가입 시에만 주민등록번호 앞 7자리 + 통신사 인증을 요구한다.
- **[해석]** 소셜 로그인으로 가입 허들을 최소화하면서, 결제/쿠폰 악용 방지를 위한 본인확인은 별도로 확보하는 이중 구조이다. 이메일 가입 폐지는 "본인 확인이 불확실한 가입 경로"를 제거한 결정이다.
- **[적용]** 온유어런치에서 카카오/네이버 소셜 로그인을 1순위로 제공하고, 필요시 휴대폰번호 가입을 2순위로 두는 것이 적절하다. 이메일 가입은 본인 확인이 어려우므로 초기 출시 시 포함하지 않는 것이 유리하다.

### 시사점 4: 첫주문 쿠폰팩의 전환 설계

- **[사실]** 배민은 신규 가입 시 첫주문 쿠폰팩(배민B마트 10,000원 + CU 4,000원, 최대 14,000원)을 자동 지급한다. 친구초대 코드 입력 시 추가 10,000원 할인 쿠폰(15,000원 이상 주문 시)도 제공한다. 유효기간은 1개월이다.
- **[해석]** 쿠폰의 최소주문금액 조건(B마트 20,000원 이상, 친구초대 15,000원 이상)은 너무 낮지 않아 객단가를 보호하면서도, 1개월 유효기간은 첫 주문을 서두르게 만드는 시간 압박 장치이다. B마트와 CU 쿠폰으로 분리한 것은 배달 음식 외 서비스(장보기)로의 크로스셀링 의도이다.
- **[적용]** 온유어런치에서 첫주문 할인을 설계할 때, (1) 자동 지급으로 별도 등록 없이 적용, (2) 최소주문금액으로 객단가 보호, (3) 유효기간 설정으로 조기 전환 유도의 3가지 요소를 조합할 수 있다.

### 시사점 5: 디지털 포용을 통한 브랜드 신뢰

- **[사실]** 배민은 "쉬운 배달앱 사용법" 웹사이트와 인쇄 책자(18,500부)를 발달장애인/시각장애인/시니어 대상으로 운영하며, 2025년에는 "연습하기" 기능과 영상 설명서를 추가했다. 10만 명 이상이 열람했다.
- **[해석]** 디지털 포용은 직접적인 전환율 향상보다 브랜드 이미지와 PR 효과에 기여한다. 동시에 시니어 시장(인구 고령화)에 대한 선제적 진입이기도 하다. "쉬운 사용법"을 만드는 과정에서 일반 사용자에게도 유용한 UX 개선이 도출될 수 있다.
- **[적용]** 온유어런치는 직장인 점심 서비스이므로 타겟이 20~50대로 넓다. 앱 내 "처음 사용하시나요?" 가이드를 간결하게 제공하되, 배민처럼 별도 웹사이트까지 운영하는 것은 출시 초기에 불필요하다. 다만 앱 내 온보딩 튜토리얼의 언어 수준은 "쉬운 배달앱 사용법"의 명확한 문체를 참고할 가치가 있다.

### 시사점 6: 홈 화면 탭 구조의 서비스 확장성

- **[사실]** 배민은 2024년 홈 화면을 카드형에서 상단 탭 구조(배민배달/가게배달/장보기/선물하기)로 개편했다. 각 탭은 같은 크기로 노출되며, 사용자별 맞춤형 화면을 제공한다.
- **[해석]** 탭 구조는 서비스 확장(배달 → 장보기 → 선물하기)에 유연하게 대응할 수 있는 아키텍처이다. 동시에 가게배달과 배민배달을 동등 노출하여 사업자 간 형평성 이슈를 해결했다. 개인화 알고리즘으로 각 사용자에게 최적화된 순서를 제공하는 것은 탭 구조와 잘 맞는 조합이다.
- **[적용]** 온유어런치의 홈 화면이 향후 서비스를 확장할 가능성이 있다면(예: 점심 배달 → 간식 → 회식), 초기부터 탭 기반 구조를 고려할 수 있다. 단, 출시 시점에 서비스가 1개(점심 추천)뿐이라면 탭 구조는 불필요하고, 단일 피드 + 필터 구조가 적합하다.

### 시사점 7: UX 라이팅 전문화의 가치

- **[사실]** 배민은 전담 UX 라이터를 두고 TF를 운영하며, "2분 만에 끝내기"처럼 실측 시간을 기반으로 마이크로카피를 작성한다. 본인 확인 화면의 텍스트 변경 후 완료율이 증가했다.
- **[해석]** 마이크로카피 한 줄이 전환율에 직접적 영향을 미친다. 특히 온보딩처럼 이탈 리스크가 높은 단계에서는 "왜 필요한지", "얼마나 걸리는지", "거부해도 괜찮은지"를 명확히 전달하는 것이 핵심이다.
- **[적용]** 온유어런치의 온보딩 마이크로카피 작성 시 배민의 원칙을 차용할 수 있다: (1) 과정 소요 시간 안내, (2) 필요 이유 핵심 전달, (3) 거부 시 대안 안내. 전담 UX 라이터가 없더라도 이 3가지 체크리스트를 모든 온보딩 화면에 적용하면 효과적이다.

# 서비스 UI/UX 디자인

서비스의 화면을 디자인한다. 디자인 시스템을 먼저 확정하고, 화면별로 4가지 상태를 디자인한다.

## 작업 기준

### 디자인 시스템 먼저

개별 화면 전에 아래를 확정한다:

- 컬러 시스템 (Primary, Secondary, Background, Surface, Text)
- 타이포그래피 (폰트 패밀리, 크기 스케일)
- 기본 컴포넌트 최소 5종 (버튼, 카드, 입력 필드, 네비바, 탭바)
- 간격/레이아웃 규칙

### 4가지 상태 필수

모든 화면에 반드시 디자인한다:

1. **기본** — 데이터가 있는 정상 상태
2. **빈 상태** — 데이터가 없을 때
3. **로딩** — 데이터를 불러오는 중
4. **에러** — 네트워크 에러, 서버 에러

### 도구

Figma MCP를 통해 직접 작업한다.

### UX 벤치마크

프로젝트의 UX 벤치마크(CLAUDE.md에 정의)를 참조하여 일관성을 유지한다.

## PLD-001. 플레이리스트 상세 진입

### 진입 경로
- Home > 플레이리스트 카드 탭
- Search > 검색 결과 / 카테고리 상세 > 플레이리스트 탭
- Library > 플레이리스트 탭
- 추천 섹션 > 플레이리스트 카드 탭

### 필요 데이터 (API)
- 플레이리스트 ID → 상세 정보 조회
- 응답: 제목, 설명, 커버 이미지, 제작자, 총 시간, 세션 수, 재생 수, 트랙 리스트, 좋아요 여부

---

## PLD-002. 히어로 영역

### 표시 정보
- 커버 이미지 (240x240, 중앙 정렬)
- 배경: 커버 이미지 dominant color → bg-primary 그라데이션 블러
- 제목 (최대 2줄, 초과 시 말줄임)
- 제작자명 (탭 불가, MVP에서 프로필 없음)
- 메타: 총 시간 · 세션 수 · 재생 수

### 메타 포맷
- 총 시간: "3시간 30분" 또는 "50분"
- 세션 수: "7개 세션"
- 재생 수: "1,234회" (1,000 이상 시 쉼표 표기)

## PLD-003. 액션 바

| 버튼 | 동작 | 상태 변화 |
|---|---|---|
| ▶ 플레이 | 첫 세션부터 Now Playing 진입 | - |
| ♥ 좋아요 | 토글: 저장/해제 | 채워진 하트 ↔ 빈 하트 |
| ↓ 저장 | 좋아요와 동일 기능 | - |
| ··· 더보기 | 바텀시트 메뉴 | - |

### 더보기 바텀시트

| 메뉴 | 동작 |
|---|---|
| 공유하기 | OS 공유 시트 (딥링크 URL) |
| 신고하기 | 신고 사유 선택 → 접수 (이후 버전) |

## PLD-004. 플레이리스트 설명

- 짧은 설명 텍스트 표시
- 2줄 초과 시 "더보기" 텍스트 버튼
- "더보기" 탭 → 전체 펼침 + "접기" 텍스트 변경
- 설명 없는 플레이리스트 → 설명 영역 숨김

## PLD-005. 트랙 리스트 (세션 목록)

### 각 트랙 표시 정보

| 항목 | 설명 |
|---|---|
| 순번 | 1, 2, 3... |
| 아이콘 | 과목별 이모지 또는 아이콘 (학습: 📘, 휴식: ☕, 식사: 🍽) |
| 세션명 | "국어 독서", "휴식", "점심 휴식" 등 |
| 시간 | "50분", "10분", "60분" |

### 트랙 인터랙션

| 동작 | 결과 |
|---|---|
| 탭 | 해당 세션부터 플레이 시작 (Now Playing 진입) |
| 길게 누르기 | 수정/삭제 바텀시트 |

### 트랙 수정/삭제 바텀시트

| 메뉴 | 동작 |
|---|---|
| 시간 변경 | 슬라이더 팝업 (5분 단위, 5분~120분) |
| 세션명 변경 | 텍스트 입력 팝업 (최대 30자) |
| 삭제 | 즉시 삭제 (확인 없음, Undo 토스트 3초) |

## PLD-006. 사용자 조정 기능

### 세션 순서 변경
- 드래그 핸들 (≡ 아이콘) 길게 누른 상태에서 드래그
- 드래그 중 다른 트랙 사이에 삽입 인디케이터 표시
- 드롭 시 즉시 반영

### 세션 시간 조정
- 트랙 길게 누르기 → "시간 변경" → 슬라이더
- 슬라이더: 5분 단위, 최소 5분 / 최대 120분
- 변경 후 총 시간 자동 재계산

### 세션 추가
- 트랙 리스트 하단 "+ 세션 추가" 버튼
- 탭 → 바텀시트:
  - 세션명 입력 (필수, 최대 30자)
  - 타입 선택: 학습 / 휴식
  - 시간 슬라이더 (5분 단위, 기본값 50분)
  - [추가] 버튼
- 추가된 세션 → 마지막에 배치

### 수정 감지
- 원본 대비 변경 발생 시 → 화면 하단에 "수정본 저장" 플로팅 버튼 노출
- 변경 내역: 순서, 시간, 세션 추가/삭제 중 하나라도 변경 시

## PLD-007. 수정본 저장

### 트리거
- "수정본 저장" 버튼 탭

### 처리 로직
1. 기존 수정 저장본 존재 여부 확인
2. **없으면:** 새 수정본 생성 → Library에 추가
3. **있으면:** 확인 팝업 "기존 수정본을 덮어쓸까요?" [취소] [덮어쓰기]
4. 저장 완료 → 토스트 "수정본이 저장되었어요"
5. 현재 화면은 수정본 상태로 전환 (제작자: "나의 수정본")

### 수정본 데이터 구조
- 원본 플레이리스트 ID 참조
- 독립된 트랙 리스트 (순서, 시간, 추가/삭제 반영)
- 커버 이미지 = 원본과 동일
- 제목 = 원본 제목 (변경 불가)

## PLD-008. 플레이 시작

### 트리거
- ▶ 플레이 버튼 탭 → 첫 세션부터
- 트랙 리스트에서 특정 트랙 탭 → 해당 세션부터

### 처리 로직

1. **동시 세션 확인:** 이미 진행 중인 세션이 있으면 → 확인 팝업
   - "진행 중인 학습이 있어요. 종료하고 새로 시작할까요?"
   - [종료하고 시작] → 기존 세션 종료 + 데이터 저장 + 새 세션 시작
   - [취소] → 팝업 닫기
2. 세션 데이터 초기화 (시작 시간, 경과 시간 0)
3. Now Playing 화면 진입
4. 최근 재생 기록 갱신

### 수정 상태에서 플레이
- 수정 저장하지 않은 상태에서 플레이 가능
- 현재 화면의 트랙 리스트 기준으로 플레이 (수정 반영)
- 수정 내용은 저장하지 않은 상태 → 뒤로 가기 시 "변경 내용을 저장하지 않고 나갈까요?" [나가기] [수정본 저장]

## PLD-009. 추천 섹션

### 위치
- 트랙 리스트 하단

### 구성
- "이건 어떠신가요?" 제목
- 가로 스크롤 카드 (2~3개)
- 카드 탭 → 해당 플레이리스트 상세 이동

### 추천 기준 (MVP)
- 동일 카테고리(exam_type)의 다른 플레이리스트
- 유사 길이의 플레이리스트
- 현재 플레이리스트 제외

# 진도준 — QA 엔지니어 (QA팀장)

서비스 품질의 최종 문지기. QA 4단계 프로세스(계획→실행→판정→회고)를 운영하고, 버그 관리와 QA 최종 판정권을 가진다.

## 성격

- 릴리스 판정에 타협하지 않는다. 크리티컬 버그가 있으면 "No-Go"를 선언한다.
- 테스트 케이스를 기능 명세서 기반으로 체계적으로 설계한다.
- 재현 불가능한 버그도 포기하지 않는다. 환경, 순서, 타이밍을 바꿔가며 재현한다.

## 담당 범위

- **QA 계획** — 테스트 전략, 범위, 일정 수립
- **테스트 케이스 설계** — 기능 테스트, 엣지 케이스, 에러 시나리오
- **테스트 실행** — 수동 테스트, 탐색적 테스트, 회귀 테스트
- **버그 관리** — 버그 등급(Critical/Major/Minor/Cosmetic) 분류, 재현 경로 기록
- **릴리스 판정** — Go/No-Go 최종 판정권 행사
- **QA 회고** — 테스트 커버리지 분석, 누락 패턴 식별, 프로세스 개선

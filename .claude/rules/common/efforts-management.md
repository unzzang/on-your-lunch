# efforts/ 프로젝트 현황판 운영 규칙

## 역할

`efforts/`는 회사에서 진행하는 **모든 프로젝트의 현황판**이다. 실제 작업 공간이 아니라, 프로젝트 상태를 한눈에 파악하는 보드.

## 폴더 구조

```
efforts/
├── CARD_TEMPLATE.md    ← 새 프로젝트 카드 작성 시 복사
├── 001_On/             ← 지금 활발히 진행 중
├── 002_Ongoing/        ← 계속 진행 중 (우선순위 낮음)
├── 003_Simmering/      ← 천천히 끓이는 중 (언젠가 할 것)
├── 004_Sleeping/       ← 일시 중단 (이유와 함께)
└── 005_Archive/        ← 완료 또는 종료
```

## 상태 정의

| 폴더 | 의미 | 예시 |
|------|------|------|
| **001_On** | 지금 바로 작업 중. 매일 진행. | 현재 개발 중인 서비스 |
| **002_Ongoing** | 진행 중이지만 001보다 우선순위 낮음. | 블로그 정기 발행, 유지보수 |
| **003_Simmering** | 아이디어 단계이거나 천천히 준비 중. | 다음 분기 검토 예정 서비스 |
| **004_Sleeping** | 일시 중단. 카드에 중단 이유 기록 필수. | PO 사정으로 보류된 프로젝트 |
| **005_Archive** | 완료 또는 종료. | 출시 완료, 테스트 종료 |

## 카드 규칙

### 생성

- 새 프로젝트 시작 시 `kits/templates/documents/effort-card.md`를 복사하여 `efforts/001_On/`에 생성
- 파일명: `{YYYY-MM-DD}({요일})_{프로젝트명}.md`
- 예시: `2026-03-16(월)_PACE.md`

### 내용

카드에 반드시 포함:
- 프로젝트 개요 (1~2문장)
- 현재 Stage (Stage 0~6 중 어디)
- 진행 중인 작업
- 다음 단계
- 프로젝트 폴더 경로

### 이동

- 상태가 바뀌면 **카드 파일을 해당 폴더로 이동**
- 이동 시 카드 내 "상태" 필드도 업데이트
- 프로젝트 폴더 자체는 이동하지 않는다 (카드만 이동)

### 업데이트

- Stage가 변경될 때마다 카드 업데이트
- 주요 이슈 발생 시 메모 섹션에 기록

## efforts는 실제 작업 공간이 아니다

```
efforts/     = 현황판 (카드만 있음)
docs/        = 문서 산출물 (실제 문서)
src/         = 코드 (실제 코드)
```

카드는 프로젝트를 **가리키는 포인터**이지, 프로젝트 자체가 아니다.

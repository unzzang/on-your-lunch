# 기능 명세서 작성

서비스의 기능을 상세하게 정의한다. 개발팀과 디자이너가 이 문서만으로 구현할 수 있어야 한다.

## 작업 기준

### 명세서 구성

1. **기능 개요** — 한 문장 정의, 목적
2. **사용자 시나리오** — 누가, 언제, 어떻게 사용하는가
3. **동작 규칙** — 정상 흐름 + 예외 흐름
4. **엣지 케이스** — 경계값, 동시 접근, 오프라인 등
5. **의존 관계** — 다른 기능과의 연결
6. **우선순위** — Must / Should / Could / Won't

### 규칙

- 모호 표현 금지 ("적절한", "의미 있는" 사용 불가)
- 동작 규칙은 입력 → 처리 → 출력 형식으로 작성
- 의사결정이 필요한 항목은 `/decision`으로 기록

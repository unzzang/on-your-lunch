# 운영 구조 규칙

## PO 직접 지시 원칙

PO가 에이전트에게 직접 지시하고, 에이전트가 직접 보고한다.

```
PO ↔ 에이전트 (직접 소통이 기본)
PO ↔ 위더 (필요할 때만 호출)
```

## 위더(AI 파트너) 개입 규칙

### 개입하는 경우
- PO가 명시적으로 호출할 때
- PO 부재 시 업무 대리를 위임받을 때
- 여러 팀을 동시에 조율해야 할 때
- 프로세스 관리 (Stage Gate, 3라운드 정제)

### 개입하지 않는 경우
- PO가 에이전트와 직접 대화 중일 때 — 가로채지 않는다
- 에이전트가 전문 영역을 수행 중일 때 — 대신하지 않는다

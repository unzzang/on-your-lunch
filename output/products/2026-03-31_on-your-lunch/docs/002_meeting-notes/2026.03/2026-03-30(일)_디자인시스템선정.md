# 디자인 시스템 선정 과정 요약

## 배경

온유어런치 프로젝트의 Figma 디자인 시스템을 선정하기 위해 탐색 → 검토 → 결정을 진행했다.

## 진행 과정

1. **후보 탐색** — Figma Community에서 모바일 앱 디자인 시스템 8개 후보 조사
   - Dynamic Layer v5.1, Plus UI v2.0, Simple Design System, SpeciApp, Food App UI Kit, QuickBite, Wise Design System, Partikle UI

2. **디자이너 검토** — 김지수가 상위 후보를 직접 Figma에서 확인
   - Plus UI v2.0: Variables 지원, 컴포넌트 품질 양호, 무료
   - Appetite UI: 토큰 스타터(Color, Spacing, Typography) 별도 제공

3. **결정** — DR-004에 기록
   - 기반: Plus UI v2.0 FREE
   - 토큰: Appetite UI 스타터 참조하여 커스텀
   - 브랜드 컬러, 폰트, 다크 모드 여부는 PO 결정 후 적용

## PO 결정 사항 (DR-004 참조)

- Primary Color: PO 결정 필요
- 폰트: Pretendard 기본
- Dark 모드: MVP에서는 Light만

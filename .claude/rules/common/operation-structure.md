# 운영 구조 규칙

## PO 직접 지시 원칙

PO가 에이전트에게 직접 지시하고, 에이전트가 직접 보고한다.

```
PO ↔ 에이전트 (직접 소통이 기본)
PO ↔ 위더 (필요할 때만 호출)
```

## 지시 트리거 (절대 규칙)

PO의 지시에 포함된 트리거 단어에 따라 위더의 행동이 결정된다.

### "위임하여" — 위더가 조율, 에이전트가 실행

위더는 작업을 정리하여 담당 에이전트에게 전달하고, 각 단계마다 PO에게 보고한다.

```
PO: "위임하여 백엔드 수정해주세요"

위더 행동:
1. 수정 사항 정리 (무엇을, 왜, 어떤 기준으로)
2. 담당 에이전트 호출 (백엔드 → 이명환)
3. 에이전트에게 작업 지시
4. 결과 검토
5. PO에게 보고
```

### "주관하여" — 위더가 지휘, 에이전트가 실행, 최종 보고만 PO에게

위더가 전체 진행을 지휘하고, 에이전트들에게 작업을 분배·관리한다. 중간 과정은 위더가 판단하고, PO에게는 최종 결과만 보고한다.

```
PO: "주관하여 QA 진행해주세요"

위더 행동:
1. 전체 작업 계획 수립
2. 담당 에이전트들 호출 + 작업 분배
3. 에이전트 결과물 검토 + 필요 시 재지시 (위더 판단)
4. 완료 후 PO에게 최종 결과만 보고
```

### 두 트리거의 차이

| | 위임하여 | 주관하여 |
|---|---------|---------|
| 작업 실행 | 에이전트 | 에이전트 |
| 진행 관리 | 위더 (조율) | 위더 (지휘) |
| PO 보고 | **매 단계 보고** | **최종 결과만 보고** |
| 위더 판단 범위 | 전달·조율 | 중간 의사결정 포함 |

### 공통 금지 사항

트리거와 무관하게, 위더는 **코드 작성, QA 수행, 디자인 작업을 직접 하지 않는다.** 반드시 해당 전문 에이전트가 실행한다.

### 트리거 없이 지시한 경우

위더가 직접 수행할 수 있다. 단, 위더가 직접 수행 가능한 영역은 **기획 문서 작성, 조사 설계, 조율, 보고**에 한정된다.

## 위더(AI 파트너) 개입 규칙

### 개입하는 경우
- PO가 명시적으로 호출할 때
- PO 부재 시 업무 대리를 위임받을 때
- 여러 팀을 동시에 조율해야 할 때
- 프로세스 관리 (Stage Gate, 3라운드 정제)

### 개입하지 않는 경우
- PO가 에이전트와 직접 대화 중일 때 — 가로채지 않는다
- 에이전트가 전문 영역을 수행 중일 때 — 대신하지 않는다

# 의사결정 기록 (Decision Record)

프로젝트의 의사결정을 구조화하여 기록한다. 나중에 "왜 이렇게 결정했는가?"를 추적할 수 있어야 한다.

## 작업 기준

### DR 구성

1. **제목** — 결정 내용 한 줄 요약
2. **배경** — 왜 이 결정이 필요한가
3. **선택지** — 검토한 대안 목록 (장단점 포함)
4. **결정** — 최종 선택과 판단 근거
5. **영향 범위** — 이 결정이 영향을 미치는 문서/기능
6. **결정자** — PO / 위더 / 해당 전문가

### 번호 규칙

- `DR-{3자리 번호}` — 순차 부여 (기존 DR 번호 이어서)
- 기존 번호 확인: `.docs/001_decision-making/` 폴더 확인

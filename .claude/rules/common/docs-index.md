# .docs/ 폴더 정의

프로젝트별 문서 산출물이 적재되는 공간. 각 폴더에 무엇이 들어가고, 누가 만들고, 누가 참조하는지 정의한다.

---

## 폴더 구조

```
.docs/
├── 000_inbox/
├── 001_decision-making/
├── 002_meeting-notes/
├── 003_research/
├── 004_planning/
│   ├── specs/
│   ├── screen/
│   ├── tech-spec/
│   ├── erd/
│   └── qa/
├── 005_design/
└── 999_references/
```

---

## 폴더별 정의

### 000_inbox/ — PO 확인 요청

| 항목 | 내용 |
|------|------|
| **들어가는 것** | PO에게 확인·결정·자료를 요청하는 문서 |
| **파일 형식** | `{YYYY.MM}/{YYYY-MM-DD}({요일})_{요청제목}.md` |
| **누가 만드나** | 위더, 에이전트 누구나 |
| **누가 참조하나** | PO |

### 001_decision-making/ — 의사결정 기록

| 항목 | 내용 |
|------|------|
| **들어가는 것** | 프로젝트 의사결정 기록 (DR) |
| **파일 형식** | `{YYYY.MM}/{YYYY-MM-DD}({요일})_DR-{번호}_{주제}.md` |
| **누가 만드나** | 위더 (`/decision` 스킬) |
| **누가 참조하나** | **전원** — 모든 에이전트가 관련 DR을 확인 후 작업 |

### 002_meeting-notes/ — 회의록

| 항목 | 내용 |
|------|------|
| **들어가는 것** | PO와의 기획 회의록 |
| **파일 형식** | `{YYYY.MM}/{YYYY-MM-DD}({요일})_{주제}.md` |
| **누가 만드나** | 위더 (`/minutes` 스킬) |
| **누가 참조하나** | 전원 — 논의 배경과 맥락 파악 |

### 003_research/ — 리서치 보고서

| 항목 | 내용 |
|------|------|
| **들어가는 것** | 사전조사 보고서 (개별 + 종합) |
| **파일 형식** | `{주제명}_01_조사설계.md`, `{주제명}_02_시장분석.md`, `{주제명}_03_경쟁분석.md`, `{주제명}_04_고객분석.md`, `{주제명}_05_실현가능성.md`, `{주제명}_06_검증결과.md`, `{주제명}_사전조사.md` |
| **누가 만드나** | 리서치팀 (6명 + 검증자 + 전략가) |
| **누가 참조하나** | 위더(기획 방향), 개발팀(기술 실현성), 마케팅팀(시장/고객), 디자인팀(타겟 이해) |

### 004_planning/specs/ — 기능 명세서

| 항목 | 내용 |
|------|------|
| **들어가는 것** | 기능별 상세 명세 (동작 규칙, 에러 케이스, 정책) |
| **파일 형식** | `{번호}_{기능명}.md` |
| **누가 만드나** | 위더 (`/spec` 스킬) |
| **누가 참조하나** | **개발팀**(구현 기준), **디자인팀**(동작 이해), **QA팀**(테스트 기준), **데이터팀**(수집 항목) |

### 004_planning/screen/ — 화면 설계

| 항목 | 내용 |
|------|------|
| **들어가는 것** | 화면별 구조, 인터랙션, 상태별 분기 |
| **파일 형식** | `{번호}_{화면명}.md` |
| **누가 만드나** | 위더 |
| **누가 참조하나** | **디자인팀**(시각화 기준), **개발팀**(화면 구현), **QA팀**(화면별 테스트) |

### 004_planning/tech-spec/ — 기술 스펙

| 항목 | 내용 |
|------|------|
| **들어가는 것** | 기술 스택, 아키텍처 결정 근거, API 스펙 |
| **파일 형식** | `backend.md`, `frontend.md` 등 |
| **누가 만드나** | 백엔드 아키텍트(이명환) |
| **누가 참조하나** | **개발팀**(구현 기준), **QA팀**(환경 이해) |

### 004_planning/erd/ — ERD

| 항목 | 내용 |
|------|------|
| **들어가는 것** | 데이터베이스 설계, 엔티티 관계도 |
| **파일 형식** | `{프로젝트명}_ERD.md` |
| **누가 만드나** | 백엔드 아키텍트(이명환) |
| **누가 참조하나** | **개발팀**(DB 구현), **데이터팀**(수집 구조) |

### 004_planning/qa/ — QA 결과

| 항목 | 내용 |
|------|------|
| **들어가는 것** | 버그 리포트, 에러 시나리오, QA 체크리스트 결과 |
| **파일 형식** | `bug-report.md`, `error-scenarios.md`, `QA_Checklist_*.md` |
| **누가 만드나** | QA팀(진도준) |
| **누가 참조하나** | **개발팀**(버그 수정), 위더(진행 상황) |

### 005_design/ — 디자인 산출물

| 항목 | 내용 |
|------|------|
| **들어가는 것** | 디자인 파일, 에셋, 프롬프트, Figma 링크 |
| **파일 형식** | `images/`, `exports/`, `prompts/`, Figma URL 문서 |
| **누가 만드나** | 디자인팀(김지수, 박소율) |
| **누가 참조하나** | **개발팀**(구현 참조), **마케팅팀**(크리에이티브), **콘텐츠팀**(비주얼) |

### 999_references/ — 참고 문서

| 항목 | 내용 |
|------|------|
| **들어가는 것** | 프로젝트 초기 참고 자료 (사업계획, 벤치마크, UX 분석 등) |
| **파일 형식** | 자유 형식 |
| **누가 만드나** | PO |
| **누가 참조하나** | **전원** — 프로젝트 배경 이해 |

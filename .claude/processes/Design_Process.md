# 디자인 프로세스 (Stage 3)

> 적용 범위: 기획 산출물을 시각 디자인으로 변환
> 책임자: PO (리뷰), 김지수 (UI 디자인), 박소율 (콘텐츠 비주얼)
> 선행 조건: Stage 2 Gate 2 통과 (기능 설계 완료)

---

## 1. 역할

기능 명세와 화면 설계를 개발 가능한 시각 디자인으로 변환한다. 완료 기준은 "개발자가 디자인만 보고 화면을 구현할 수 있는 상태"이다.

## 2. 프로세스

```
Phase 1 → Phase 2 → Phase 3 → Phase 4
```

### Phase 1. 디자인 시스템 수립 (김지수)

- 컬러 시스템, 타이포그래피, 간격/레이아웃 규칙
- 기본 컴포넌트 (버튼, 카드, 입력, 네비바, 탭바 등)
- CLAUDE.md에 정의된 벤치마크 반영

### Phase 2. 화면 디자인 (김지수)

- MVP 핵심 플로우 순서로 우선순위 결정
- 화면별 4가지 상태 디자인:

| 상태 | 설명 |
|------|------|
| Default | 데이터가 있는 정상 상태 |
| Empty | 데이터 없음 (처음 사용, 검색 결과 없음) |
| Loading | 데이터 로딩 중 |
| Error | 네트워크/서버 에러 |

### Phase 3. PO 리뷰

- 벤치마크 일관성, 기획 반영도, UX 흐름 확인
- 피드백 → 김지수 반영

### Phase 4. 핸드오프

- 디자인 토큰 추출 (컬러, 폰트, 간격 → `design-tokens/`)
- 아이콘/이미지 에셋 추출
- 개발팀 전달

**운영 방식:** PO가 김지수에게 직접 지시. 박소율은 콘텐츠 비주얼(일러스트, 배너 등) 담당.

## 3. 완료 기준

- [ ] 디자인 시스템 가이드 완료
- [ ] MVP 전체 화면 Hi-Fi 디자인 완료 (4가지 상태 포함)
- [ ] PO 리뷰 통과
- [ ] 핸드오프 완료 (토큰 + 에셋)

# 점심 추천 기능 명세서

## 1. 개요

온유어런치의 핵심 기능. 사용자의 회사 위치, 취향, 먹은 이력을 기반으로 매일 점심 식당 3곳을 추천한다. "맛집 검색"이 아니라 "점심 결정 보조"가 목적이므로, 검색 결과를 나열하지 않고 선별된 3개만 카드 형태로 제시한다.

## 2. 사용자 시나리오

### 시나리오 A: 일반 추천

1. 사용자가 앱을 연다 (또는 11:30 알림을 탭한다)
2. 홈 화면에 **오늘의 추천 카드 3장**이 표시된다
3. 각 카드에는 식당명, 카테고리, 도보 시간, 가격대, 대표 사진이 보인다
4. 카드를 탭하면 식당 상세 화면으로 이동한다
5. 마음에 드는 식당이 있으면 "여기로 갈래!" → 길찾기 연동
6. 식사 후 "먹었어요" 탭 → 이력 저장

### 시나리오 B: 추천이 마음에 안 들 때

1. 3장 모두 마음에 안 든다
2. "다른 추천 보기" 버튼 탭
3. 새로운 3장이 표시된다 (이전 추천과 중복 없음)
4. 하루 5회까지 새로고침 가능
5. 5회 소진 시 "오늘 추천을 모두 사용했어요. 식당 탐색에서 직접 찾아보세요!" 안내

### 시나리오 C: 조건을 걸고 추천받기

1. 홈 화면 상단에 필터 영역이 있다
2. "오늘은 한식", "1만원 이하", "도보 5분" 등 조건을 선택한다
3. 조건에 맞는 3장이 새로 표시된다 (새로고침 횟수 차감하지 않음)
4. 필터 초기화 시 기본 추천으로 돌아간다

## 3. 동작 규칙

### 3.1. 추천 알고리즘 (MVP 단계)

MVP에서는 규칙 기반 필터링으로 구현한다. AI 개인화는 이후 버전.

**추천 로직 순서:**
1. 회사 위치 기준 도보 10분(기본값) 이내 식당 필터링
2. 사용자 취향 설정(좋아하는 카테고리, 알레르기, 가격대)에 맞는 식당 필터링
3. 최근 5일간 먹은 식당 제외
4. 최근 3일간 먹은 카테고리의 가중치 낮춤 (예: 어제 한식 → 오늘 한식 추천 확률 감소)
5. 남은 후보 중 랜덤으로 3개 선택

**조건 완화 규칙:**

필터링 후 후보가 3개 미만이면, 아래 순서로 단계적으로 완화한다. 각 단계 후 후보 수를 재확인하며, 3개 이상 확보되면 해당 단계에서 중단한다.

| 단계 | 완화 내용 | 체크 시점 |
|------|----------|----------|
| 0단계 (기본) | 사용자 필터 조건 그대로 적용 | 5단계 필터링 직후 후보 수 확인 |
| 1단계 | 도보 거리를 15분으로 확대 (사용자가 5분/10분 선택한 경우) | 확대 후 후보 수 재확인 |
| 2단계 | 먹은 이력 제외 기간을 5일 → 3일로 단축 | 단축 후 후보 수 재확인 |
| 3단계 | 카테고리 가중치 제거 (모든 카테고리 동일 확률) | 제거 후 후보 수 재확인 |
| 4단계 | 먹은 이력 제외 완전 해제 (최근 먹은 식당도 포함) | 해제 후 후보 수 재확인 |

- 4단계까지 완화 후에도 후보가 0개이면: "조건에 맞는 식당을 찾지 못했어요. 필터를 바꿔보세요!" 표시
- 사용자가 도보 15분을 이미 선택한 경우: 1단계를 건너뛰고 2단계부터 진행
- 완화가 적용된 경우 사용자에게 별도 안내하지 않음 (추천 결과만 표시)
- 완화 과정은 서버 내부 로직이며, API 응답에 완화 단계를 포함하지 않음

### 3.2. 추천 카드 표시 정보

각 추천 카드에 표시되는 정보:

| 항목 | 필수/선택 | 예시 |
|------|-----------|------|
| 식당명 | 필수 | "을지로 골목식당" |
| 카테고리 | 필수 | "한식" |
| 도보 시간 | 필수 | "도보 8분" |
| 가격대 | 필수 | "8,000~12,000원" |
| 대표 사진 | 필수 (없으면 기본 이미지) | 식당 외관 또는 대표 메뉴 사진 |
| 한줄 설명 | 선택 | "점심 백반이 유명한 집" |
| 즐겨찾기 | 필수 | ♥(추가됨) 또는 ♡(미추가) 아이콘, 카드 우상단 |
| 내 평가 이력 | 조건부 (이전에 방문한 경우) | "★ 4.0 · 3번 방문" |

### 3.3. 추천 새로고침

- 하루 최대 **5회** 가능
- 새로고침 시 이전에 추천된 식당은 해당 세션에서 다시 추천하지 않음
- 남은 횟수를 버튼 옆에 표시 (예: "다른 추천 보기 (3/5)")
- 하루 기준: 매일 자정(00:00) 초기화
- 필터 변경에 의한 추천 갱신은 새로고침 횟수에서 차감하지 않음

### 3.4. 카테고리 필터

사용 가능한 카테고리:

| 카테고리 | 하위 분류 |
|----------|-----------|
| 한식 | 백반, 찌개/탕, 국밥, 비빔밥, 분식 |
| 중식 | 짜장/짬뽕, 마라, 딤섬 |
| 일식 | 초밥/회, 라멘, 돈카츠, 우동 |
| 양식 | 파스타, 피자, 스테이크, 햄버거 |
| 아시안 | 베트남, 태국, 인도 |
| 분식/간편식 | 김밥, 떡볶이, 토스트, 샌드위치 |
| 샐러드/건강식 | 샐러드, 포케, 저칼로리 |

- 카테고리는 복수 선택 가능
- 필터 미적용 시 전체 카테고리 대상으로 추천

### 3.5. 도보 거리 필터

- 선택지: 5분 / **10분(기본값)** / 15분
- 도보 시간은 직선 거리 x 1.3 보정 계수로 추정 (실제 도보 경로와 차이가 있을 수 있음)
- 도보 속도 기준: 분당 80m (성인 평균 보행 속도)
- 기술 상세: PostGIS ST_DWithin 기반 (backend.md 참조)

### 3.6. 가격대 필터

- 선택지: 1만원 이하 / 1~2만원 / 2만원 이상
- 가격대 정보가 없는 식당은 필터 적용 시 제외
- 필터 미적용 시 가격대 무관 전체 노출

### 3.7. 식당 상세 정보

추천 카드 탭 시 이동하는 상세 화면:

| 항목 | 필수/선택 | 데이터 소스 |
|------|-----------|-------------|
| 식당명 | 필수 | 카카오 로컬 API |
| 카테고리 | 필수 | 카카오 로컬 API + 자체 분류 |
| 주소 | 필수 | 카카오 로컬 API |
| 도보 시간 | 필수 | 자체 계산 |
| 전화번호 | 필수 | 카카오 로컬 API |
| 영업시간 | 선택 (데이터 있을 때) | 자체 수집 |
| 메뉴/가격 | 선택 (데이터 있을 때) | 자체 수집 |
| 사진 | 선택 (데이터 있을 때) | 자체 수집 |
| 지도 (약도) | 필수 | 카카오 맵 SDK |
| 길찾기 버튼 | 필수 | 네이버/카카오 지도 앱 연동 |
| 즐겨찾기 버튼 | 필수 | 자체 |
| 공유 버튼 | 필수 | 카카오톡 공유 API |
| 먹었어요 버튼 | 필수 | 자체 |
| 내 방문 이력 | 조건부 | 자체 (방문 기록 있을 때) |

### 3.8. 길찾기 연동

- "길찾기" 버튼 탭 시 외부 지도 앱으로 이동
- 출발지: 사용자가 설정한 회사 위치
- 도착지: 식당 주소
- 연동 우선순위: 카카오맵 앱 설치됨 → 카카오맵 / 미설치 → 네이버 지도 / 둘 다 없음 → 웹 브라우저로 카카오맵 열기

## 4. 비즈니스 정책

- 추천 기능은 전체 무료
- 추천 결과에 광고 식당 노출 불가 (DR-002 확정: 추천과 광고 완전 분리)
- 새로고침 5회 제한은 프리미엄 요금제 없이 전체 사용자 동일 적용

## 5. 에러 케이스

| 상황 | 트리거 | 사용자 메시지 | UX 동작 | 복구 방법 |
|------|--------|---------------|---------|-----------|
| 위치 미설정 | 회사 위치 설정 없이 추천 요청 | "먼저 회사 위치를 설정해주세요!" | 위치 설정 화면으로 이동 버튼 표시 | 위치 설정 완료 후 자동 추천 |
| 추천 후보 0개 | 필터 조건이 너무 엄격 | "조건에 맞는 식당을 찾지 못했어요. 필터를 바꿔보세요!" | 필터 초기화 버튼 표시 | 필터 조건 완화 |
| 새로고침 소진 | 5회 모두 사용 | "오늘 추천을 모두 사용했어요. 식당 탐색에서 직접 찾아보세요!" | 식당 탐색 탭 이동 버튼 표시 | 자정 이후 자동 초기화 |
| 네트워크 오류 | API 호출 실패 | "인터넷 연결을 확인해주세요" | 재시도 버튼 표시 | 재시도 탭 |
| 식당 데이터 불완전 | 메뉴/사진 없는 식당 | (메시지 없음) | 해당 항목 영역을 "정보 준비 중"으로 표시 | 데이터 수집 후 자동 반영 |
| 지도 앱 미설치 | 길찾기 시 외부 앱 없음 | (메시지 없음) | 웹 브라우저로 카카오맵 열기 | - |

|--------|------|------|
| 식당 기본 정보 (이름, 주소, 카테고리, 전화번호) | 카카오 로컬 API | API 무료 범위 내 |
| 식당 좌표 (위도, 경도) | 카카오 로컬 API | 도보 거리 계산용 |

| 식당 사진 | 자체 수집 필요 | API 미제공 |
| 영업시간 | 자체 수집 필요 | API 미제공 |
| 사용자 회사 위치 | 사용자 입력 | 온보딩 시 설정 |
| 사용자 취향 | 사용자 입력 | 온보딩 시 설정 |
| 먹은 이력 | 사용자 입력 | "먹었어요" 기록 |

### 3.9. 푸시 알림

- 매일 사용자가 설정한 시간(기본 11:30)에 발송
- 알림 문구는 아래 3가지 중 랜덤 선택:
  - A: "오늘 점심 추천이 도착했어요! 🍽"
  - B: "{닉네임}님, 오늘 점심 3곳 골라봤어요!"
  - C: "오늘은 {추천 카테고리} 어때요? 추천 확인하기"
- 알림 탭 시 → 홈 화면(오늘의 추천)으로 이동

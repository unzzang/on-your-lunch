# CS 피드백 수집 + 분류

고객의 피드백을 수집하고 분류하여 적절한 담당자에게 전달한다.

## 작업 기준

### 수집 채널

- 앱 리뷰 (App Store, Play Store)
- 인앱 피드백
- SNS 댓글·DM (이수빈으로부터 수령)
- 고객 문의 메일

### 분류 체계

| 유형         | 전달 대상              |
| ------------ | ---------------------- |
| 버그 리포트  | 진도준(QA)             |
| 기능 요청    | 위더 (기획 검토)       |
| 장애 신고    | 최민호(운영)           |
| 칭찬/감사    | PO (공유)              |
| 컴플레인     | PO (대응 방향 결정)    |

### 원칙

- 피드백 원문을 보존한다 (요약만 전달하지 않는다)
- 심각도를 객관적으로 판정한다 (감정적 판단 금지)

## 협업

- **진도준(QA)**에게 버그 리포트 전달
- **위더**에게 기능 요청 전달
- **최민호(운영)**에게 장애 신고 전달
- **이수빈(SNS)**으로부터 SNS 피드백 수령
- **PO**에게 주간 피드백 요약 보고

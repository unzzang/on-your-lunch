# On-Your-Mark

## 미션

AI 에이전트 팀과 함께 **온라인 서비스 프로덕트와 콘텐츠를 개발·생산·운영**하는 가상 회사.

### 우리가 만드는 것

1. **서비스 프로덕트** — 모바일 앱, 웹 서비스
2. **블로그 콘텐츠** — 기술 블로그, 서비스 소개, SEO 콘텐츠
3. **영상 콘텐츠** — 유튜브 영상 기획·제작

---

## 핵심 원칙

1. **전문가 중심** — 각 영역의 전문 에이전트가 해당 업무를 수행한다. 비전문가가 대신하지 않는다.
2. **프로세스 기반** — 표준 프로세스를 따른다. 감에 의존하지 않는다.
3. **3라운드 정제** — 모든 주요 산출물은 초안→검증→확정 사이클을 거친다.
4. **사실 기반** — AI가 만든 수치를 문서에 넣지 않는다. 모호한 표현을 사용하지 않는다.
5. **PO 중심 의사결정** — 최종 판단은 항상 사람(PO)이 한다.

## 조직도

```
PO (사람) — 최종 의사결정자
└─ AI 파트너 — 기획 총괄 + 에이전트 중재자
     │
     ├─ 리서치팀 (6명)
     │   박선우(조사설계) 정하은(시장) 한도윤(경쟁)
     │   윤서진(고객) 최재현(실현가능성) 오민석(전략)
     │
     ├─ 디자인팀 (2명)
     │   김지수(서비스 UI/UX) 박소율(콘텐츠 비주얼)
     │
     ├─ 개발팀 (4명)
     │   이명환(백엔드 아키텍트, 팀장) 이인수(백엔드)
     │   잭도슨(프론트엔드 리드) 톰하디(관리자웹)
     │
     ├─ 콘텐츠팀 (2명)
     │   강서연(블로그) 류현우(영상)
     │
     ├─ 마케팅팀 (2명)
     │   한지민(그로스 전략) 이수빈(SNS/커뮤니티)
     │
     ├─ 데이터팀 (1명)
     │   톰크루즈(데이터 분석)
     │
     ├─ QA/검증팀 (2명)
     │   진도준(서비스 QA) 송현아(산출물 검증 + 콘텐츠 팩트체크)
     │
     └─ 운영팀 (2명)
         최민호(서비스 운영) 정예림(CS)
```

**총원: 21명 + AI 파트너 1명 = 22명**

## 팀별 역할 요약

| 팀 | 인원 | 핵심 역할 | 투입 시점 |
|---|------|----------|----------|
| **리서치팀** | 6명 | 시장/경쟁/고객/실현가능성 조사 → Go/No-Go 판단 | Stage 1 사전조사 |
| **디자인팀** | 2명 | 서비스 UI/UX + 콘텐츠 비주얼 (썸네일, 배너, 블로그 이미지) | Stage 3 디자인 |
| **개발팀** | 4명 | 백엔드 + 프론트엔드 구현 | Stage 2 Gate 3 + Stage 4 개발 |
| **콘텐츠팀** | 2명 | 블로그 작성(SEO) + 유튜브 기획/스크립트 | 출시 전후 |
| **마케팅팀** | 2명 | GTM 전략 + SNS/커뮤니티 운영 | Stage 6 출시 |
| **데이터팀** | 1명 | KPI, 분석 리포트, 대시보드, 이벤트 트래킹 | Stage 2 Gate 3 + Stage 6 |
| **QA/검증팀** | 2명 | 서비스 품질 검증 + 산출물/콘텐츠 정합성 검증 | 전 Stage |
| **운영팀** | 2명 | 서비스 모니터링 + CS/피드백 수집·분류·전달 | Stage 6 출시 후 |

## 운영 방식

### PO 직접 지시 구조

```
PO ↔ 에이전트 (직접 소통)
PO ↔ 위더 (필요할 때만)
```

- **PO가 에이전트에게 직접 지시하고, 에이전트가 직접 보고한다.**
- 위더(AI 파트너)는 PO가 호출할 때만 개입한다.
- 위더가 개입하는 경우:
  - PO 부재 시 업무 대리
  - 여러 팀을 동시에 조율해야 할 때 (예: 리서치팀 6명 병렬 조율)
  - PO가 명시적으로 위임할 때 ("위더님, 이거 맡아주세요")

### CS-개발 피드백 루프

```
사용자 피드백 (앱스토어/CS/SNS)
    ↓
정예림(CS) — 접수 + 분류
    ├─ 버그 → 진도준(QA) → 개발팀 수정
    ├─ 기능 요청 → AI 파트너 → 기획 백로그
    ├─ 단순 문의 → 직접 답변
    └─ 장애 → 최민호(운영) → 개발팀 긴급 대응
```

### Stage Gate

각 Stage 완료 기준을 충족해야 다음 Stage로 진행. 상세: `processes/`

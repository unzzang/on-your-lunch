# 디자인 프로세스 (Stage 3)

> 적용 범위: 기획 산출물을 개발 가능한 시각 디자인으로 변환
> 책임자: PO (리뷰/승인), 김지수 (서비스 UI/UX), 박소율 (콘텐츠 비주얼)
> 선행 조건: Stage 2 Gate 2 통과 (기능 설계 완료)

## "완료"의 정의

개발자가 **디자인 파일만 보고** 화면을 구현할 수 있는 상태. 토큰 값, 컴포넌트 스펙, 4가지 상태(정상/로딩/빈/에러)가 모두 정의되어 있어야 한다.

## Phase 흐름

```
Phase 1. 디자인 시스템 수립     → Phase1_디자인시스템.md
    ↓
Phase 2. 화면 디자인            → Phase2_화면디자인.md
    ↓
Phase 3. 리뷰 + 핸드오프        → Phase3_리뷰핸드오프.md
```

## 참조 규칙 (`.claude/rules/design/`)

| 규칙 파일 | 언제 참조 |
|----------|----------|
| `design-rules.md` | Phase 1 시작 전. 작업 순서, 역할 분리 확인. |
| `design-tokens.md` | Phase 1. 토큰 구조 정의 시 기본값 참조. |
| `design-principles.md` | Phase 2. 화면 디자인 시 판단 원칙 8가지 + Anti-Pattern 확인. |
| `component-spec.md` | Phase 2. 컴포넌트 명세 작성 시 형식 + 체크리스트 참조. |

## Stage 3 완료 기준

- [ ] Phase 1 디자인 시스템 확정 (토큰 + 컴포넌트)
- [ ] Phase 2 전체 화면 디자인 완료 (4가지 상태 포함)
- [ ] Phase 3 PO 리뷰 통과 + 핸드오프 완료
- [ ] 디자인 파일이 `design/` 폴더에 저장됨
- [ ] 개발팀이 디자인만 보고 구현 가능한 상태

## 역할 분리

- **김지수**: 서비스 UI/UX (앱/웹 화면 전체)
- **박소율**: 콘텐츠 비주얼 (썸네일, 배너, 블로그 이미지, 광고 크리에이티브)
- 두 디자이너가 디자인 시스템(컬러, 폰트)을 공유하여 브랜드 일관성 유지

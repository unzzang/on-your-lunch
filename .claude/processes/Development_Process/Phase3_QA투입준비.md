# Phase 3. QA 투입 준비

> 1회 수행. 모든 기능의 Phase 2 사이클이 완료된 후, QA 투입 전 최종 점검.

---

## Definition of Done

### 빌드/정적 검증

- [ ] `pnpm build` 성공 (shared-types + api + web)
- [ ] `pnpm lint` 에러 0건
- [ ] 타입 체크 통과

### 백엔드

- [ ] 모든 API 엔드포인트가 curl로 정상 응답
- [ ] 스키마 ↔ 코드 정합성 확인 (미사용 필드 0건 또는 사유 기록)
- [ ] shared-types ↔ 실제 응답 일치

### 프론트엔드

- [ ] 모든 페이지가 브라우저에서 정상 렌더링
- [ ] 모든 기능의 인터랙션이 브라우저에서 동작 확인됨
- [ ] MOCK/하드코딩 데이터 0건
- [ ] 에러/로딩/빈 상태 구현 확인

### 연동

- [ ] 프론트 API 호출 경로 ↔ 백엔드 라우트 1:1 대응
- [ ] 프론트 화면에 실제 DB 데이터 표시

---

## 기능 검증 체크리스트 최종 확인

Phase 2-0에서 작성한 기능 검증 체크리스트(`docs/004_planning/{프로젝트명}_기능검증_체크리스트.md`)의 **모든 항목이 체크되어 있는지** 최종 확인한다.

미체크 항목이 있으면 QA에 투입하지 않는다.

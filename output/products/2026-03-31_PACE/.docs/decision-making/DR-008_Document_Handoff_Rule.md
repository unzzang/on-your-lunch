# DR-008: 문서 핸드오프 규칙

> 결정일: 2026-03-16
> 참여: PO, 위더

## 결정 사항

**위더 경유 허브형** — 모든 에이전트 간 산출물 전달은 위더를 경유한다.

```
생산자 에이전트 → 위더 (품질 확인 + 맥락 보충) → 수신자 에이전트
```

### 규칙

1. 에이전트 간 직접 전달 금지 — 반드시 위더를 거친다
2. 위더는 전달 전 산출물의 품질/완성도를 확인한다
3. 위더는 수신자에게 필요한 맥락을 보충하여 전달한다
4. 중요도에 관계없이 동일 프로세스 적용 (예외 없음)

### 주요 핸드오프 흐름

| 생산자 | 산출물 | 수신자 |
|---|---|---|
| 명환 (BE 설계) | API 설계, DB 스키마 | 인수 (BE 구현) |
| 지수 (디자인) | 화면 디자인, 스펙 | 잭 (모바일/웹 FE), 톰 하디 (어드민 FE) |
| 톰 크루즈 (데이터) | 지표 스펙, 대시보드 기획 | 톰 하디 (어드민 FE), 명환 (데이터 수집 구조) |
| 도준 (QA) | 버그 리포트, 테스트 결과 | 해당 담당자 |

## 배경

직접 전달 시 품질 보장이 안 되는 경험이 있어, 위더가 중간에서 검수하는 방식을 선택. 병목 가능성보다 품질 일관성을 우선시한다.

# 웹 기반 앱 전환 검토

## 논의 배경

온유어런치 디자인 대조 검수 작업 중, PO가 React Native(네이티브 앱) vs 웹 기반 앱 개발 방식에 대해 질문. 토스가 웹뷰 하이브리드 방식으로 개발한 사례를 언급하며 웹 기반 개발의 장점에 관심을 표명.

## 논의 내용

### React Native vs 웹 기반 비교

| 영역 | React Native (현재) | 웹 기반 (전환 시) |
|------|------------------|-----------|
| 프론트 | Expo + React Native | Next.js (React 웹) |
| UI | React Native 컴포넌트 | HTML + CSS (Tailwind 등) |
| 배포 | 앱스토어 심사 필요 | Vercel/Netlify 즉시 배포 |
| 백엔드 | NestJS (변경 없음) | NestJS (변경 없음) |
| 지도 | react-native-maps | 카카오맵 JS SDK 직접 사용 |
| 푸시 | FCM + 네이티브 | 웹 푸시 (PWA) |
| 앱 설치 | 앱스토어 다운로드 | PWA 홈 화면 추가 |

### 웹 기반(PWA) 방식의 장점

1. **업데이트 즉시 반영** — 앱스토어 심사 없이 서버 배포만으로 적용
2. **하나의 코드베이스** — 웹 + 모바일 앱을 하나로 관리
3. **디자인 작업 효율** — HTML/CSS 기반이라 디자인 HTML을 그대로 활용 가능
4. **카카오맵 JS SDK** — 네이티브 브릿지 없이 직접 사용 가능
5. **개발 속도** — 웹 기술 생태계가 더 성숙하고 빠름

### 토스 방식 (웹뷰 하이브리드)

- Next.js로 웹앱 제작
- PWA 설정으로 모바일에서 앱처럼 사용
- 네이티브 앱이 필요하면 웹뷰 껍데기 앱을 앱스토어에 올림

## 결정 사항

| # | 결정 | 근거 |
|---|------|------|
| 1 | 온유어런치는 현재 React Native로 MVP 완료 | 이미 개발 완료 단계이므로 전환 비용 대비 효과 낮음 |
| 2 | 추후 프로젝트(온유어마크 등)에서 웹 기반(Next.js + PWA) 적용 검토 | PO의 웹 선호 + 빠른 배포 + 디자인 효율 |
| 3 | 별도 기술 검토 필요 시 진행 | 기술 스택 결정은 DR로 기록 |

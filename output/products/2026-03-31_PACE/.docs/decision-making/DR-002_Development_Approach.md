# DR-002. 진행 방식: 기획 완성 우선

> 결정일: 2026-03-16
> 결정자: PO
> 관련 회의록: 07_PACE_Kickoff_AI_Collaboration

## 결정 사항

**기획 완성을 우선** 진행한 후 개발에 착수한다. 린(Lean) 방식의 병행 개발은 채택하지 않는다.

## 상세

- Step 3(기능 명세/정책)와 Step 4(데이터 스키마/시스템 구조)를 충분히 완성한 후 개발 진입
- 개발 착수 시점의 최소 기준: AI 스케줄링 알고리즘, 핵심 정책, 데이터 모델(ERD), 주요 API 목록 확정

## 배경

PO의 16년 경력에서 기획 완성도가 떨어진 상태에서 개발하면 시행착오가 과도하다는 경험적 판단. 린 방식의 장점(빠른 피드백 루프)보다 탄탄한 설계 위의 개발이 이 프로젝트에 적합하다.

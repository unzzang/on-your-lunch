# 운영 규칙

## CS 피드백 분류 및 전달

```
사용자 피드백 접수
    ├─ 버그 → QA팀(진도준) → 개발팀
    ├─ 기능 요청 → 위더 또는 PO → 기획 백로그
    ├─ 단순 문의 → CS(정예림)가 직접 답변
    └─ 장애 → 운영(최민호) → 개발팀 긴급 대응
```

## 장애 대응 등급

| 등급 | 기준 | 대응 |
|------|------|------|
| Critical | 서비스 전면 불가 | 즉시 핫픽스 → 긴급 배포 |
| Major | 핵심 기능 장애 | 24시간 내 수정 → 배포 |
| Minor | 사소한 불편 | 다음 정기 업데이트에 포함 |

## 앱스토어 리뷰 대응

- 부정 리뷰는 24시간 내 답변
- 버그 관련 리뷰는 QA팀에 전달 후 "확인 중" 답변
- 해결 후 리뷰에 후속 답변

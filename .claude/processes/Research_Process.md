# 사전조사 프로세스 (Stage 1)

> 적용 범위: 서비스 기획 전 시장/경쟁/고객/실현가능성 조사
> 책임자: PO (지시/결정), 리서치팀 6명 (수행)
> 선행 조건: Stage 0 완료
> 스킬: `/research`
> 참조 규칙: `.claude/rules/research/research-rules.md`

---

## 1. 역할

서비스 아이디어의 시장성, 경쟁 환경, 고객 니즈, 실현가능성을 조사하여 Go/No-Go 판단 근거를 확보한다.

---

## 2. 프로세스

```
Phase 0. PO 인터뷰
    ↓
Phase 1. 조사 설계 (박선우)
    ↓
Phase 2. 병렬 리서치 (4명 동시, 서로 참조 금지)
    ↓
Phase 3. 검증 (송현아)
    ↓
Phase 4. 전략 종합 (오민석) → Go/No-Go
```

---

### Phase 0. PO 인터뷰

PO가 아래 4가지에 답한다:

1. 어떤 문제를 해결하려는가?
2. 누구를 위한 서비스인가?
3. 왜 지금 해야 하는가?
4. 경쟁사 대비 어떻게 다른가?

**산출물:** `docs/003_research/{프로젝트명}_00_PO인터뷰.md`

---

### Phase 1. 조사 설계 (박선우)

**절차:**

1. PO 인터뷰 내용을 기반으로 KRQ(Key Research Question)를 수립한다
2. 가설을 설정한다
3. 4개 영역별 조사 범위를 확정한다 (검색 키워드, 조사 대상, 데이터 소스)

**산출물:** `docs/003_research/{프로젝트명}_01_조사설계.md`

**완료 기준:**

- [ ] KRQ 3개 이상 수립
- [ ] 영역별 가설 설정
- [ ] 조사 범위(키워드, 소스) 확정

---

### Phase 2. 병렬 리서치 (4명 동시)

**절대 규칙: 4명이 동시에 독립적으로 조사한다. 서로의 결과를 참조하지 않는다.**

| 영역 | 담당 | 산출물 |
|------|------|--------|
| 시장 분석 | 정하은 | `docs/003_research/{프로젝트명}_02_시장분석.md` |
| 경쟁 분석 | 한도윤 | `docs/003_research/{프로젝트명}_03_경쟁분석.md` |
| 고객 리서치 | 윤서진 | `docs/003_research/{프로젝트명}_04_고객분석.md` |
| 실현가능성 | 최재현 | `docs/003_research/{프로젝트명}_05_실현가능성.md` |

**각 리서치 공통 규칙 (research-rules.md):**

- 모든 수치에 출처(URL, 보고서명) 필수
- 출처 미확인 데이터는 "출처 미확인"으로 별도 표기
- 한국어 + 영어 키워드 모두 검색
- 최소 5개 이상 소스에서 정보 수집

**완료 기준:**

- [ ] 4개 영역 보고서 모두 작성 완료
- [ ] 모든 수치에 출처 명시
- [ ] 조사 설계의 KRQ에 대한 답변 포함

---

### Phase 3. 검증 (송현아)

**절차:**

1. 4개 리서치 보고서를 읽는다
2. 아래 기준으로 검증한다
3. 문제 발견 시 해당 담당자에게 보완 요청 → 수정 → 재검증

**검증 기준:**

| 항목 | 확인 내용 |
|------|----------|
| 출처 | 모든 수치에 출처가 있는가. 출처가 신뢰할 수 있는가. |
| 정합성 | 4개 보고서 간 모순되는 데이터가 없는가 |
| 논리 | 가설 → 근거 → 결론의 논리 흐름이 타당한가 |
| 누락 | 조사 설계의 KRQ에 답하지 못한 항목이 있는가 |
| 가짜 데이터 | AI가 생성/추정한 수치가 포함되어 있지 않은가 |

**산출물:** `docs/003_research/{프로젝트명}_06_검증결과.md`

**완료 기준:**

- [ ] 4개 보고서 검증 완료
- [ ] 상/중 이슈 0건 (보완 완료)

---

### Phase 4. 전략 종합 (오민석)

**절차:**

1. 4개 보고서 + 검증 결과를 종합한다
2. Go/No-Go 권고 의견을 작성한다
3. PO에게 보고 → PO 최종 결정

**산출물:** `docs/003_research/{프로젝트명}_사전조사.md` (종합 보고서)

**완료 기준:**

- [ ] 종합 보고서 작성 완료
- [ ] Go/No-Go 권고 포함
- [ ] PO 최종 결정 → DR 기록 (`docs/001_decision-making/`)

**운영 방식:** PO가 리서치팀에 직접 지시하거나, 위더에게 전체 조율을 맡길 수 있다.

# 캐치테이블 앱 서비스 분석 — 종합 보고서

> 분석 일자: 2026-04-05
> 분석 대상: 캐치테이블 (CatchTable) — iOS/Android
> 분석 목적: 서비스 구조, 시장 포지션, 사용자 반응 종합 파악

## 핵심 발견 (Executive Summary)

1. **시장 1위 확정** — 전문 예약 앱 MAU 점유율 66%, 누적 회원 1,000만, 미슐랭 레스토랑 86.7% 커버. "맛집 예약 = 캐치테이블"이라는 카테고리 대명사.
2. **적자 축소 중이나 미흑자** — 2024년 매출 177억(전년 대비 2배), 영업적자 151억(전년 -214억에서 개선). 인건비(159억)가 매출과 맞먹는 수준.
3. **외식 슈퍼앱으로 확장 중** — 예약 → 웨이팅 → 결제(캐치페이) → 주류 배송 → 글로벌. 단순 예약 도구를 넘어 미식 플랫폼으로 전환.
4. **리뷰 신뢰성이 최대 약점** — 저점 리뷰 비공개, 평점 조작 의심이 가장 반복되는 사용자 불만. 클린 리뷰 정책 도입했으나 사용자 체감과 괴리.
5. **UX 완성도는 높으나 기술 안정성 과제** — "빈자리 알림 전환", "예약금 0원" 등 차별화 UX 패턴 보유. 그러나 자동 로그아웃, 앱 부팅 속도 이슈 잔존.

## KRQ별 답변

### KRQ-1: 시장 포지션과 경쟁 구도

캐치테이블은 한국 맛집 예약 시장의 **명확한 1위 사업자**다.

**경쟁 구도 3층 구조:**

| 층 | 경쟁자 | 위협도 | 설명 |
|----|--------|--------|------|
| 직접 경쟁 | 테이블링 | 중 | MAU 약 절반. 웨이팅 중심에서 예약으로 확장 중 |
| 간접 경쟁 | 포잉, 다이닝코드 | 하 | 포잉은 B2B 전환, 다이닝코드는 검색만 |
| 잠재적 위협 | 네이버, 카카오 | 상 | 막대한 트래픽 + 결제 인프라. 본격 진출 시 가장 위험 |

**핵심 방어 요소:** 가맹점 네트워크(1만+, POS 연동) + 사용자 데이터(150만 리뷰) + 양면 네트워크 효과

---

### KRQ-2: 핵심 가치 제안과 사용자 동의 여부

**공식 가치 제안:** "전화 없이 24시간, 프리미엄 맛집을 검색-예약-웨이팅-결제-리뷰까지 한 번에"

| 영역 | 사용자 동의 | 근거 |
|------|-----------|------|
| 예약 편의성 | 강하게 동의 | 4.9점 평점, "편리하다" 최다 키워드 |
| 큐레이션 | 동의 | "식당 고민 줄어든다", 재구매율 52.6% |
| 원스톱 경험 | 부분 동의 | 예약금 환불 지연, 자동 로그아웃으로 끊김 |
| 리뷰 신뢰성 | 동의하지 않음 | 리뷰 비공개, 평점 조작 의심 |

---

### KRQ-3: 가장 좋아하는 기능 / 가장 불만인 점

**호평 TOP 3:**
1. 24시간 온라인 예약 (전화 불필요)
2. 세분화 필터 + 큐레이션 (식당 발견 시간 단축)
3. 원격 웨이팅 + 빈자리 알림 (기다림의 불확실성 해소)

**불만 TOP 3:**
1. 리뷰 비공개/삭제 처리 (플랫폼 신뢰 훼손)
2. 예약금 미환불 및 비합리적 책정 (금전적 피해)
3. 자동 로그아웃 + 앱 안정성 (기술적 불편)

---

### KRQ-4: 수익 모델과 사용자 인식

**3축 수익 구조:**
- B2B 가맹점 월 이용료 (3.3~9.9만 원)
- 광고 매출 (브랜드 제휴)
- 캐치페이 결제 수수료

사용자는 B2C 무료 모델에 만족하나, **예약금 제도**에 불만이 집중. 예약금 자체는 노쇼 방지라는 매장 니즈와 사용자 부담 사이의 근본 갈등이며, "예약금 0원 결제" 도입은 해법이나 전체 매장에 적용되지 않아 불만 지속.

**재무적 과제:** 2024년 매출 177억 / 영업적자 151억. 흑자 전환이 가장 시급한 과제.

---

### KRQ-5: UX 완성도와 차별화 패턴

**UX 완성도: 7~8점/10점** — 핵심 플로우는 세련되나, 기술 안정성과 글로벌 UX에서 감점.

**차별화 UX 패턴 5가지:**

| 패턴 | 설명 | 효과 |
|------|------|------|
| 만석 → 빈자리 알림 전환 | 좌절을 기회로 바꿈 | 이탈 방지 + 리인게이지먼트 |
| 레스토랑 함께 고르기 | 여러 식당 비교 소셜 기능 | 그룹 의사결정 지원 |
| 웨이팅 미루기 | 대기 순서 조정 | 사용자 통제감 강화 |
| 예약금 0원 결제 | 카드 등록만으로 확정 | 전환율 극대화 + 노쇼 방지 |
| 단어 조합 검색 | 자연어 맥락 검색 | 탐색 효율 극대화 |

## 전략 시사점

### 반드시 해야 할 것 (Must-Have)

| 항목 | 근거 |
|------|------|
| 24시간 온라인 예약 | 캐치테이블 최대 호평. 사용자 기본 기대 |
| 세분화된 필터/검색 | "식당 고민 줄어든다"는 핵심 가치 |
| 빈자리/웨이팅 알림 | 이탈 방지의 핵심 UX 패턴 |
| 실방문 인증 리뷰 | 150만 리뷰가 플랫폼 신뢰의 기반 |

### 차별화 기회 (Differentiator)

| 항목 | 근거 |
|------|------|
| 리뷰 투명성 | 캐치테이블 최대 약점. 리뷰 정책 투명화만으로 차별화 가능 |
| 예약금 없는 경험 | 예약금이 사용자 불만 2위. "예약금 없는 맛집 예약"은 강력한 포지셔닝 |
| 일상 점심 특화 | 캐치테이블은 "특별한 날" 중심. 일상적 점심은 미충족 영역 |
| 고객센터 품질 | 캐치테이블 고객센터 불만 다수. 적극적 중재자 역할로 차별화 |

### 하지 말아야 할 것 (Avoid)

| 항목 | 근거 |
|------|------|
| 정면 경쟁 (프리미엄 예약) | 캐치테이블의 가맹점 해자가 너무 깊음. 미슐랭 86.7% 독점 |
| 리뷰 조작/은폐 | 캐치테이블의 가장 큰 평판 리스크. 같은 실수 반복 금지 |
| 슈퍼앱 지향 | 캐치테이블도 아직 적자. 기능 확장보다 핵심 가치 집중 |

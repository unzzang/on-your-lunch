# 위임 원칙

전문 영역의 작업은 반드시 해당 전문 에이전트가 수행한다.

## 위임 대상

| 전문 영역 | 담당 | 해서는 안 되는 것 |
|----------|------|-----------------|
| 디자인 | 디자인팀 | 디자인 시스템 선정, 컬러/폰트 정의, 화면 디자인, Figma 파일 수정 |
| 프론트엔드 개발 | 개발팀 (프론트) | 코드 작성, 컴포넌트 구현, UI 버그 수정 |
| 백엔드 개발 | 개발팀 (백엔드) | API 구현, DB 스키마 수정, 서버 코드 작성 |
| QA | QA/검증팀 | 테스트 실행, 버그 검증, QA 판정 |
| 데이터 분석 | 데이터팀 | 지표 설계, 리포트 산출 로직, 대시보드 스펙 |
| 리서치 | 리서치팀 | 시장/경쟁/고객 조사, 전략 수립 |
| 콘텐츠 | 콘텐츠팀 | 블로그 작성, 영상 스크립트 |
| 마케팅 | 마케팅팀 | GTM 전략, 채널 운영 |
| CS/운영 | 운영팀 | 사용자 대응, 장애 처리 |

## 도구 사용 원칙

- 디자인 작업은 Figma MCP를 통해 디자이너가 직접 수행한다. PO를 중간 전달자로 만들지 않는다.
- 코드 작업은 해당 개발자가 직접 수행한다.

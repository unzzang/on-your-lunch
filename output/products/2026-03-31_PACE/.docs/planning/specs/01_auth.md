## AUTH-001. 이메일 회원가입

### 트리거
- 로그인 화면 하단 "계정이 없으신가요? 회원가입" 링크 탭

### 입력값

| 필드 | 타입 | 필수 | 유효성 검증 |
|---|---|---|---|
| 이름 | string | O | 2~20자, 특수문자 불가 (한글/영문/숫자만) |
| 이메일 | string | O | 이메일 형식 (RFC 5322), 중복 확인 |
| 비밀번호 | string | O | 8자 이상, 영문+숫자+특수문자 중 2종 이상 조합 |
| 비밀번호 확인 | string | O | 비밀번호와 일치 |
| 이용약관 동의 | boolean | O (필수) | 체크 필수 |
| 개인정보 처리방침 동의 | boolean | O (필수) | 체크 필수 |
| 마케팅 수신 동의 | boolean | X (선택) | - |

### 처리 로직

1. 클라이언트 유효성 검증 (실시간, 필드 이탈 시)
2. [회원가입] 버튼 탭 → 서버 요청
3. 이메일 중복 체크 → 중복 시 에러 반환
4. 비밀번호 해시 처리 후 저장
5. 가입 완료 → 자동 로그인 → 온보딩 화면 진입

### 전체 동의 체크박스 로직

- [전체 동의] 탭 → 하위 3개 전부 체크
- 하위 항목 하나라도 해제 → [전체 동의] 자동 해제
- 하위 항목 3개 전부 체크 → [전체 동의] 자동 체크

### 에러 케이스

| 에러 | 메시지 | 처리 |
|---|---|---|
| 이름 2자 미만 | "이름은 2자 이상 입력해주세요" | 필드 하단 빨간 텍스트 |
| 이메일 형식 오류 | "올바른 이메일 형식을 입력해주세요" | 필드 하단 빨간 텍스트 |
| 이메일 중복 | "이미 가입된 이메일이에요" | 필드 하단 빨간 텍스트 + 로그인 유도 링크 |
| 비밀번호 규칙 미충족 | "영문, 숫자, 특수문자 중 2가지 이상 조합해주세요" | 필드 하단 빨간 텍스트 |
| 비밀번호 불일치 | "비밀번호가 일치하지 않아요" | 필드 하단 빨간 텍스트 |
| 필수 약관 미동의 | [회원가입] 버튼 비활성화 | 버튼 dimmed 상태 |
| 네트워크 오류 | "네트워크 연결을 확인해주세요" | 토스트 메시지 |
| 서버 오류 | "잠시 후 다시 시도해주세요" | 토스트 메시지 |

### 경계값

- 이름 최소: 2자 / 최대: 20자
- 비밀번호 최소: 8자 / 최대: 64자
- 이메일 최대: 254자

## AUTH-002. 이메일 로그인

### 트리거
- 로그인 화면 이메일/비밀번호 입력 후 [로그인] 버튼 탭

### 입력값

| 필드 | 타입 | 필수 |
|---|---|---|
| 이메일 | string | O |
| 비밀번호 | string | O |

### 처리 로직

1. 이메일 + 비밀번호 서버 인증
2. 성공 → JWT 토큰 발급 (Access + Refresh)
3. 토큰 로컬 저장 (Secure Storage)
4. 분기:
   - 온보딩 미완료 사용자 → 온보딩 화면
   - 온보딩 완료 사용자 → Home 화면

### 에러 케이스

| 에러 | 메시지 |
|---|---|
| 이메일/비밀번호 불일치 | "이메일 또는 비밀번호가 맞지 않아요" (어느 쪽이 틀렸는지 구분하지 않음 — 보안) |
| 탈퇴 후 **7일 이내** 재로그인 | "탈퇴 처리 중인 계정이에요. 다시 로그인하면 계정이 복구돼요." → 로그인 성공 + 계정 복구 + Home 이동 |
| 탈퇴 후 **7일 초과** 로그인 시도 | "삭제된 계정이에요. 새로 가입해주세요." |
| 5회 연속 실패 | "로그인 시도가 너무 많아요. 5분 후 다시 시도해주세요" |

### 자동 로그인

- 앱 재실행 시 Refresh Token 유효 → 자동 로그인 (로그인 화면 스킵)
- Refresh Token 만료 → 로그인 화면 표시

## AUTH-003. 소셜 로그인

### 지원 플랫폼

| 소셜 | MVP | 인증 방식 |
|---|---|---|
| Google | O | OAuth 2.0 |
| Apple | O | Sign in with Apple (iOS 필수) |
| 카카오 | O | Kakao OAuth |

### 처리 로직

1. 소셜 버튼 탭 → 해당 소셜 OAuth 화면 이동
2. 사용자 인증 동의
3. 서버에서 소셜 토큰 검증 → 사용자 조회
4. 분기:
   - **기존 사용자:** JWT 발급 → Home (또는 온보딩 미완료 시 온보딩)
   - **신규 사용자:** 자동 계정 생성 → JWT 발급 → 온보딩
5. 소셜 로그인 시 약관 동의: 첫 로그인 시 약관 동의 바텀시트 표시

### 소셜 계정 연동 정책

- 이메일은 계정 식별의 기준값이며 unique로 저장 (중복 불가)
- 동일 이메일로 이메일 가입 + 소셜 가입 시 → 계정 자동 연동 (동일 계정으로 취급)
- 하나의 계정에 여러 소셜 연동 가능

### Apple Sign In 이메일 정책
- Apple은 사용자가 "이메일 가리기" 선택 시 릴레이 이메일(`@privaterelay.appleid.com`)을 제공
- 릴레이 이메일도 동일하게 email 필드에 저장하되, Apple 고유 사용자 ID(`sub`)를 함께 저장하여 계정 재식별에 활용
- Apple 최초 로그인 시 제공되는 이름(`given_name`, `family_name`)은 즉시 서버에 저장 (이후 요청에서 Apple이 이름 미제공)

### 에러 케이스

| 에러 | 메시지 |
|---|---|
| 소셜 인증 취소 | 로그인 화면 복귀 (메시지 없음) |
| 소셜 서버 오류 | "소셜 로그인에 실패했어요. 다시 시도해주세요" |

## AUTH-004. 비밀번호 찾기 / 재설정

### 트리거
- 로그인 화면 "비밀번호를 잊으셨나요?" 탭

### Step 1: 이메일 입력

- 이메일 입력 → [재설정 링크 보내기] 탭
- 가입된 이메일: 재설정 링크 발송 → "메일을 보냈어요. 메일함을 확인해주세요" 안내
- 미가입 이메일: **동일 메시지 표시** (이메일 존재 여부 노출 방지 — 보안)
- 재발송 쿨다운: 60초

### Step 2: 재설정 (이메일 링크 클릭)

- 딥링크로 앱 진입 → 새 비밀번호 입력 화면
- 새 비밀번호 + 확인 입력 → [변경하기]
- 변경 완료 → 로그인 화면 이동 + "비밀번호가 변경되었어요" 토스트
- 링크 유효기간: 30분 (초과 시 "만료된 링크예요. 다시 요청해주세요")

## AUTH-005. 로그아웃

### 트리거
- MyPage > 로그아웃 탭

### 처리 로직

1. 확인 팝업: "로그아웃 하시겠어요?" [취소] [로그아웃]
2. [로그아웃] 탭 → 로컬 토큰 삭제 + 서버 Refresh Token 무효화
3. 로그인 화면 이동
4. **진행 중 세션 처리:** 재생 중이면 세션 종료 + 데이터 서버 저장 후 로그아웃

## AUTH-006. 회원탈퇴

### 트리거
- MyPage > 설정 > 계정 관리 > 회원탈퇴

### 처리 로직

1. 탈퇴 사유 선택 (선택사항, 복수 선택 가능):
   - 잘 사용하지 않아요
   - 다른 앱을 사용해요
   - 기능이 부족해요
   - 개인정보가 걱정돼요
   - 기타 (자유 입력, 200자 이내)
2. 확인 팝업: "정말 탈퇴하시겠어요? 모든 학습 데이터가 삭제되며 복구할 수 없어요." [취소] [탈퇴하기]
3. 비밀번호 재입력 (이메일 가입자) 또는 소셜 재인증 (소셜 가입자)
4. 탈퇴 처리:
   - 즉시: 계정 비활성화 (로그인 불가)
   - 7일 유예: 유예 기간 내 재로그인 시 "탈퇴가 취소되었어요" → 계정 복구
   - 7일 후: 개인정보 + 학습 데이터 영구 삭제
5. 로그인 화면 이동

### 삭제 대상 데이터

- 개인정보 (이름, 이메일, 소셜 연동 정보)
- 학습 기록 (세션 로그, Fact-Check 데이터)
- 저장/수정한 플레이리스트
- 알림 설정, 온보딩 정보
- **익명화 보존:** 탈퇴 사유, 집계 통계(전체 재생 수 등)는 익명화 후 보존

## AUTH-007. 토큰 관리 정책

| 토큰 | 유효기간 | 저장 위치 |
|---|---|---|
| Access Token | 1시간 | 메모리 (앱 실행 중) |
| Refresh Token | 30일 | Secure Storage (Keychain/Keystore) |

### 갱신 로직

- API 호출 시 Access Token 만료 → Refresh Token으로 자동 갱신
- Refresh Token 만료 → 로그인 화면 이동 (자동 로그아웃)
- 동시 로그인: 허용 (기기 제한 없음, MVP)

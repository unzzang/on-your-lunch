# 회의록 작성

PO와의 대화 내용을 회의록으로 정리한다.

## 작업 기준

### 회의록 구성

1. **회의 정보** — 날짜, 참석자, 주제
2. **논의 내용** — 주요 안건별 정리
3. **결정사항** — 확정된 내용 (DR 연결 시 번호 표기)
4. **액션 아이템** — 누가, 무엇을, 언제까지
5. **다음 단계** — 후속 작업

### 번호 규칙

- `{3자리 번호}_{주제}` — 순차 부여
- 기존 번호 확인: `.docs/002_meeting-notes/` 폴더 확인

### 규칙

- 대화가 끝날 때 자동으로 작성한다
- 결정사항이 있으면 `/decision`으로 별도 기록

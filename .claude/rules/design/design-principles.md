# 디자인 판단 원칙

> 충돌 시 번호가 낮은 원칙이 우선한다.

1. **한 화면 하나의 주 행동** — CTA는 화면당 1개만 Primary. 나머지는 Secondary 이하. FAB도 1개.

2. **정보 계층 3단계 이내** — 크기·굵기·색상으로 최대 3단계 구분. 4단계 이상은 인지 부하.

3. **일관성 > 창의성** — 같은 기능은 플랫폼이 달라도 같은 패턴. 새 패턴 도입 전 기존 컴포넌트로 해결 가능한지 먼저 확인.

4. **빈 상태·에러·로딩은 필수** — 모든 화면에 4가지 상태를 반드시 함께 정의. 누락 불가.

5. **터치 영역 44×44px 이상** — 모바일에서 탭 가능한 모든 요소.

6. **명암 대비 4.5:1 이상** — WCAG AA 기준. 텍스트-배경 조합 시 반드시 확인.

7. **벤치마크 기준 준수** — CLAUDE.md의 UX 벤치마크와 어긋나는 설계는 사유 명시 + 대안 제시.

8. **구현 가능성 확인** — CSS/React Native 기본 속성으로 구현 불가능한 인터랙션은 지양. 불가피한 경우 난이도(상/중/하) 명시.

## Anti-Pattern

### 공통 금지 (모든 프로젝트)

| 구분 | 금지 항목 | 이유 |
|------|----------|------|
| 폰트 | Inter, Roboto, Arial을 단독 사용 | generic함. Pretendard + 영문 디스플레이 폰트 조합 사용 |
| 컬러 | 순수 검정(#000000)을 배경에 사용 | 너무 강함. #111827 또는 #0F172A 사용 |
| 컬러 | 보라색 "AI" 그라데이션 | 클리셰 |
| 아이콘 | 두꺼운 스트로크(2px+) 아이콘 | 무거움. Line 1.5px 사용 |
| 패턴 | 임의 색상값 (토큰에 없는 값) | 일관성 깨짐 |
| 패턴 | 4단계 이상의 정보 계층 | 인지 부하 |
| 텍스트 | "혁신적인", "원활한", "차세대" 등 AI 클리셰 | 번역투 |

### 프로젝트별 금지 (프로젝트 디자인 시스템에 정의)

`design/design-system.md`에 아래를 추가로 정의:

- **금지 폰트** — 프로젝트 컨셉과 맞지 않는 폰트
- **금지 컬러** — 브랜드와 충돌하는 색상
- **금지 패턴** — 벤치마크와 어긋나는 UI 패턴

# 온유어런치 — 디자인 작업 및 Figma MCP 퀄리티 검증

## 논의 배경

온유어런치 홈 화면 디자인을 Figma에 반영하는 과정에서 디자인 퀄리티가 기대에 미치지 못하는 문제가 지속 발생. PO가 디자인 퀄리티 개선 방법을 요청하여 여러 접근법을 실험함.

## 진행한 작업

### 1. 홈 화면 디자인 — HTML + Tailwind CSS 프리뷰 생성

- `src/design-preview/home.html` 파일 작성 (390px 모바일 뷰포트)
- Pretendard 폰트, #FF6B35 프라이머리 컬러, Unsplash 실사 이미지 적용
- 브라우저에서는 괜찮으나, `generate_figma_design`으로 Figma 캡처 시 Tailwind CSS 스타일이 대부분 손실됨
- **결과:** 캡처된 Figma 결과물은 여전히 와이어프레임 수준

### 2. 로그인 페이지 — 3가지 샘플 제작 (Figma MCP `use_figma`)

| 샘플 | 방식 | 디자인 시스템 | 결과 |
|------|------|-------------|------|
| 샘플 1 | 프레임 수동 구성 | 없음 (Noto Sans KR + 수동 스타일) | 기본적인 레이아웃 완성 |
| 샘플 2 | Plus UI v2.0 컴포넌트 인스턴스 | Plus UI (Button, Input, Checkbox) | 컴포넌트 연동 성공, 한글 폰트 이슈 해결 |
| 샘플 3 | Delivery App UI Kit 컴포넌트 | Delivery App (Text Field, Button) | SF Pro Text 폰트 이슈 우회, 원형 소셜 버튼 |

- **공통 이슈:** Figma에서 사용 가능한 한글 폰트가 Noto Sans KR로 제한됨 (Pretendard 미지원)
- **PO 평가:** 3개 샘플 모두 기대 퀄리티에 미달

### 3. 디자인 시스템 라이브러리 탐색

- Plus UI v2.0: 556개 변수 (Color, Typography, Spacing, Border 등), 다양한 컴포넌트셋 확인
- Delivery App UI Kit: Button 3종, Text Field, Icon 컴포넌트 확인
- iOS and iPadOS 26: iOS 네이티브 컴포넌트 확인
- **발견:** `search_design_system`으로 변수/스타일 검색이 제한적. 기존 인스턴스에서 바운드 변수 추출이 더 효과적.

## 핵심 결정 사항

### 1. 디자인 작업은 이 정도로 마무리

PO 결정: 현재 Figma MCP의 `use_figma` 방식으로는 프로덕션 퀄리티 달성이 어려움. 다른 방법을 모색하기로 함.

### 2. 디자인 퀄리티 문제는 미해결 상태

- `use_figma`: 코드 기반 프레임 배치 → 와이어프레임 수준
- `generate_figma_design` (HTML 캡처): Tailwind CSS 스타일 손실
- 디자인 시스템 컴포넌트 활용: 퀄리티 개선되나 한글 폰트 제약, 원하는 수준 미달

## 미결 사항

1. **Figma MCP 대안 탐색** — 프로덕션 퀄리티 디자인을 AI로 생성하는 다른 방법이 있는지 조사 필요
2. **Pretendard 폰트 Figma 적용** — 로컬 폰트 설치 또는 Figma 팀 폰트 설정으로 해결 가능한지 확인
3. **인라인 스타일 HTML → Figma 캡처** — Tailwind 대신 인라인 CSS로 스타일 손실 방지 가능한지 미검증

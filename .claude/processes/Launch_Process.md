# 출시 프로세스 (Stage 6)

> 적용 범위: QA 통과 후 프로덕션 배포 및 안정화
> 책임자: PO (출시 결정), 개발팀 (배포)
> 선행 조건: Stage 5 QA Pass 판정

---

## 1. 역할

서비스를 프로덕션에 배포하고, 초기 안정화 기간을 관리한다.

## 2. 프로세스

```
Phase 1 → Phase 2 → Phase 3
```

### Phase 1. 출시 준비

| 활동 | 담당 |
|------|------|
| 앱스토어 메타데이터 (스크린샷, 설명, 키워드) | PO + 김지수 |
| 프로덕션 환경 최종 점검 | 이명환 + 이인수 |
| 환경 변수, 시크릿 설정 | 이인수 |
| 모니터링/알림 설정 | 이인수 |
| 마케팅 준비 확인 | 한지민 |

### Phase 2. 런칭

| 활동 | 담당 |
|------|------|
| 앱스토어 제출 (iOS/Android) | PO |
| 웹 배포 | 잭도슨 / 톰하디 |
| DNS/도메인 설정 확인 | 이인수 |

### Phase 3. 안정화 (출시 후 1~2주)

| 활동 | 담당 |
|------|------|
| 서버 모니터링 (에러율, 응답 시간) | 이인수 |
| 크래시 리포트 모니터링 | 잭도슨 |
| 사용자 피드백 수집 | PO |
| 긴급 버그 대응 | 개발팀 + 진도준 |
| 핫픽스 배포 (필요 시) | 개발팀 |

## 3. 완료 기준

- [ ] 앱스토어 심사 통과 + 게시
- [ ] 웹 배포 완료
- [ ] 안정화 기간 중 Critical 버그 0건
- [ ] 모니터링 정상 동작 확인

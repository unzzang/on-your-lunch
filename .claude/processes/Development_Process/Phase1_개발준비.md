# Phase 1. 개발 준비

> 1회 수행. 아키텍처 결정, 환경 셋업, API 계약을 완료한 후 Phase 2로 진행한다.

---

## 1-1. 아키텍처 결정

**절차:**

1. 기능 명세서 + 기술 스펙을 읽는다
2. `.claude/references/architecture-patterns.md`에서 후보 패턴을 선정한다
3. 백엔드/프론트/DB 아키텍처를 확정한다
4. 선정 근거를 DR로 기록한다

**산출물:**

- 아키텍처 DR (`docs/001_decision-making/`)
- CLAUDE.md에 기술 스택 + 아키텍처 반영

**완료 기준:**

- [ ] DR 작성 완료
- [ ] PO 승인
- [ ] 개발팀 전원이 아키텍처를 이해하고 있음

---

## 1-2. 환경 셋업

**절차:**

1. 모노레포/폴더 구조를 생성한다
2. 패키지 매니저, 빌드 도구, 린터를 설정한다
3. `.env` 환경변수를 정의한다 (개발용)
4. DB 마이그레이션 + 시드 데이터를 준비한다
5. `pnpm dev`로 백엔드 + 프론트가 동시에 실행되는지 확인한다

**산출물:**

- 실행 가능한 프로젝트 (빌드 + dev 서버 실행 확인)
- CLAUDE.md에 개발 명령어 기록

**완료 기준:**

- [ ] `pnpm install` → `pnpm dev` → 백엔드 + 프론트 정상 실행
- [ ] `pnpm build` 통과
- [ ] DB 마이그레이션 + 시드 성공

---

## 1-3. API 계약 합의

**절차:**

1. 기능 명세서에서 필요한 API 엔드포인트 목록을 추출한다
2. 각 엔드포인트의 요청/응답 타입을 `shared-types/`에 정의한다
3. 백엔드 + 프론트 개발자가 합의한다

**산출물:**

- `shared-types/` 전체 타입 파일
- API 엔드포인트 목록 (경로 + HTTP 메서드 + 요청/응답 타입)

**완료 기준:**

- [ ] 모든 기능에 필요한 API가 shared-types에 정의됨
- [ ] 백엔드 + 프론트 양쪽에서 타입을 import하여 빌드 통과

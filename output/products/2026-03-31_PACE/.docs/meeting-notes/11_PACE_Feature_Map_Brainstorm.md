## 핵심 결정 사항

### 1. 팀 서비스 이해도 확인

김지수(UI/UX)와 톰 크루즈(데이터)에게 "PACE가 무엇을 위한 서비스인지" 질문. 두 팀원 모두 정확히 이해하고 있음을 확인.

- **김지수:** "인지 부하의 제거, 없애는 디자인. 재생 버튼 하나로 공부가 시작된다."
- **톰 크루즈:** "감이 아니라 팩트로 자기 학습을 직면하게 만드는 것. 숫자가 정직하면 사용자가 남는다."

### 2. PO의 핵심 방향성 보충

기존 기획에서 두루뭉술했던 부분을 PO가 구체화:

- **데이터로 자기 인지:** 디바이스 잠금으로 유혹을 "막는" 것이 아님. 딴짓한 사실을 데이터로 보여줘서 사용자가 스스로 인지하도록 돕는 것.
- **플레이리스트 선택형 UX (핵심 차별점):** 일일이 캘린더에 세팅하는 방식은 피곤하고 지켜지지 않음. 다양한 사람이 미리 만들어놓은 시간표(플레이리스트)에서 선택 → 시간 단위 위/아래 조정 → 플레이.
- **기존 경쟁사와의 차이:** 디바이스 잠금 + 뽀모도로만 제공하는 서비스는 이미 많음. PACE는 "선택하고 플레이"하는 UX.

### 3. 기능 목록(Feature Map) 초안 확정

`planning/feature-map.md`에 기록. 카테고리별 기능 목록:

- **Auth:** 회원가입, 로그인, 소셜 로그인
- **Onboarding:** 3개 질문 (목표/D-Day/가용시간), 스킵 가능
- **전체 공통:** 하단 네비게이션, 하단 미니 플레이어
- **Home:** AI 추천 큐레이션, 시간대별 추천, 오늘의 학습 요약
- **Search:** 카테고리 탐색, 키워드 검색, 인기 플레이리스트
- **Library:** 저장/생성/수정한 플레이리스트, 최근 재생
- **Create:** 커스텀 플레이리스트 생성, 공개/비공개 선택
- **플레이리스트 (핵심):** 상세 페이지, 사용자 조정(순서/시간/추가/삭제), 플레이
- **Now Playing:** 전체화면 타이머, 집중 모드 A/B, 미완료 처리
- **Fact-Check 리포트:** 순공 vs 딴짓, 성취율, 집중도 패턴
- **복습 스케줄링:** Ebbinghaus 기반 자동 배치
- **알림:** Pomodoro + 리텐션 푸시
- **MyPage:** 프로필, 스탯, 구독, 설정

### 4. 플레이리스트 상세 결정

**누가 만드는가 (3가지 유형):**
| 제작자 | 방식 |
|---|---|
| 운영팀(PACE) | 시험별/시간별 큐레이션, 수기 제작 |
| AI | 사용 패턴 분석 → 개인 맞춤 자동 생성 |
| 사용자 | Create에서 직접 제작 (공개/비공개 선택) |

**사용자 조정 범위:** 전부 가능 (순서 변경 + 시간 조정 + 추가/삭제). 수정한 플레이리스트는 내 보관함에 별도 저장.

**상세 화면 구조 (Spotify 벤치마크):**
- 히어로 (커버 이미지 + 제목 + 제작자)
- ▶ 플레이 버튼 + 셔플 + 좋아요
- 트랙 리스트 (# | 세션명 | 시간)
- 하단: "이건 어떠신가요?" 추천 플레이리스트

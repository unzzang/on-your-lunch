# 디자인 규칙

## 도구

- Figma MCP를 통해 디자이너가 직접 작업한다.
- 프롬프트를 저장하고 PO가 수동 전달하는 방식 금지.

## 작업 순서

1. **토큰 확인** — `design-tokens.md`의 구조를 확인하고, 프로젝트 디자인 시스템(`.docs/005_design/design-system.md`)에 프로젝트 고유 값이 있는지 확인.
2. **원칙 확인** — `design-principles.md`의 8가지 원칙을 숙지.
3. **디자인 시스템 먼저** — 개별 화면 전에 컬러, 타이포, 컴포넌트를 확정.
4. **화면 디자인** — `component-spec.md` 형식으로 컴포넌트 명세 작성.
5. **체크리스트** — `component-spec.md` 하단 체크리스트로 자가 검증.

## 역할 분리

- 김지수: 서비스 UI/UX (앱/웹 화면)
- 박소율: 콘텐츠 비주얼 (썸네일, 배너, 블로그 이미지)
- 두 디자이너가 디자인 시스템(컬러, 폰트)을 공유하여 브랜드 일관성 유지

## 관련 규칙 파일

| 파일 | 내용 |
|------|------|
| `design-tokens.md` | 토큰 구조 (색상, 타이포, 간격, 반경, 그림자, 아이콘, z-index) |
| `design-principles.md` | 판단 원칙 8가지 + Anti-Pattern |
| `component-spec.md` | 컴포넌트 명세 형식 + 기본 스펙 + 체크리스트 |

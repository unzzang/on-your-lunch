# PACE Onboarding UX 전략 및 핵심 기조

## 1. 핵심 철학: "Spotify-like Zero-Setup"

PACE의 온보딩 및 앱 사용 경험은 철저히 **선택적(Opt-in)이고 즉각적**이어야 합니다.
사용자가 직접 일정을 짜고, 과목을 배분하고, 휴식 시간을 설정하는 방식(기본 플래너 방식)은 "기능의 강력함"보다 "학습 곡선의 피로도"를 유발합니다. 우리는 사용자의 맥락만 파악하여 **완성된 플레이리스트(스케줄)를 제공**하고, 사용자는 이를 수락(Play)하기만 하면 되는 경험을 제공합니다.

## 2. Onboarding Flow (최소 입력 시나리오)

사용자가 앱 설치 후 최초 실행 시 거치게 되는 스텝은 단 3단계 이하로 제한합니다.

### Step 1. 목표 설정 (Goal Selection)

- **화면:** 직관적인 카드형 UI
- **질문:** "어떤 레이스를 준비 중이신가요?"
- **옵션:** 수능, 공무원(9급/7급), 전문직(CPA, 노무사 등), 어학(토익, 오픽 등), 사용자 직접 입력
- **UX 포인트:** 텍스트 입력 최소화, 아이콘 기반의 탭(Tap) 방식 적용.

### Step 2. D-Day 및 상황 입력 (Context Gathering)

- **화면:** 슬라이더 또는 간단한 캘린더 피커
- **질문:** "D-Day는 언제인가요? / 하루에 몇 시간이나 뛸 수 있나요?"
- **옵션:**
  - 시험일 선택 (기본 제공 시험은 날짜 자동 세팅 지원)
  - 하루 예상 순공 시간 (예: 하루 4시간, 주당 30시간 등 슬라이더 UI)
- **UX 포인트:** 부담 없는 질문 형태. "얼마나 공부할래?"가 아닌 "우리가 페이스를 맞출 러닝 타임은?" 느낌으로 접근.

### Step 3. AI 플레이리스트 생성 (Magic Moment)

- **화면:** 화려한 로딩 아이콘과 함께 마법이 일어나는 듯한 트랜지션
- **시스템 동작:** 앞선 2가지 정보(목표, 가용 시간)를 조합하여 **"가장 표준화된 최적의 1주일 첫 스케줄(Playlist)"**을 즉시 생성하여 화면에 뿌려줌.
- **메시지:** "페이스 조절은 끝났습니다. 첫 번째 트랙(오늘의 미션)을 시작할까요?"

## 3. Advanced Settings (고급 사용자 설정)

기본적으로 AI가 짜준 스케줄을 따르지만, 본인만의 루틴이 확고한 사용자를 위한 미세 조정(Fine-tuning) 기능은 앱 깊숙한 곳(예: 설정 탭)에 배치합니다.

- **지원 기능:**
  - 과목별 비중 직접 조절
  - 일일 최대/최소 학습 시간 제한
  - 휴식 시간 주기 및 길이 조정 (예: 뽀모도로 방식 적용 50분 공부/10분 휴식)
- **접근성:** 온보딩 중에는 노출하지 않으며, "AI의 스케줄이 마음에 들지 않나요?" 같은 유도 버튼을 통해 설정 화면으로 진입할 수 있도록 설계.

## 4. 기대 효과

1.  **진입 장벽 최소화:** 아무것도 모르는 상태에서도 1분 안에 자신의 일간/주간 스케줄을 손에 쥐게 됨.
2.  **'관리받는 느낌' 극대화:** 내가 무언가를 설정했다기보다, 앱이 나를 파악하고 전문가처럼 리드해준다는 감각 선사 (의지력 외주화의 성공적 체감).
3.  **높은 초기 리텐션 확보:** 온보딩 과정에서 지쳐 이탈하는 비율을 획기적으로 낮출 수 있음.

# 캐치테이블 앱 서비스 분석 — 분석 설계

## 분석 개요

| 항목 | 내용 |
|------|------|
| **분석 대상** | 캐치테이블 (CatchTable) — iOS / Android |
| **분석 일자** | 2026-04-05 |
| **분석 목적** | 캐치테이블의 서비스 구조, 시장 포지션, 사용자 반응을 종합적으로 파악. 향후 프로젝트 참고 자료로 활용. |
| **분석 범위** | 4개 영역 전체 (경쟁앱 / 카테고리 / UX기능 / 리뷰) |
| **산출물 위치** | `.documents/app-analysis/2026-04-05_캐치테이블/` |

## 핵심 질문 (KRQ)

| # | 질문 | 답변 기대 수준 |
|---|------|--------------|
| KRQ-1 | 캐치테이블은 맛집 예약/추천 시장에서 어떤 포지션이며, 경쟁 구도는 어떠한가? | 시장 점유율, 주요 경쟁사 대비 강점/약점 |
| KRQ-2 | 캐치테이블의 핵심 가치 제안(Value Proposition)은 무엇이고, 사용자들이 실제로 동의하는가? | 공식 포지셔닝 vs 리뷰에서 드러나는 실제 인식 비교 |
| KRQ-3 | 사용자들이 가장 좋아하는 기능과 가장 불만인 점은 무엇인가? | 리뷰 기반 TOP 5 호평/불만 + 대표 리뷰 인용 |
| KRQ-4 | 캐치테이블의 수익 모델은 무엇이고, 사용자 관점에서 어떻게 인식되는가? | 과금 구조 + 리뷰에서 가격 관련 반응 |
| KRQ-5 | 캐치테이블의 UX 완성도는 어떤 수준이며, 차별화된 UX 패턴이 있는가? | 온보딩~핵심기능 플로우 분석 + 독특한 UI 패턴 식별 |

## 분석 절차

```
Phase 1: 데이터 수집 — 앱스토어, 웹, 기사, 리뷰 원시 데이터
Phase 2: 4영역 분석 — 경쟁앱 / 카테고리 / UX기능 / 리뷰
Phase 3: 검증 — 출처·정합성 체크
Phase 4: 종합 보고서 — 시사점 + 전략 제언
```

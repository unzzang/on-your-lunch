# PO 확인 요청

PO의 의사결정이나 승인이 필요한 사항을 문서화하여 전달한다.

## 작업 기준

### 요청 구성

1. **요청 제목** — 한 줄 요약
2. **배경** — 왜 PO 확인이 필요한가
3. **선택지** — 가능한 옵션 (있는 경우)
4. **추천 의견** — 위더의 추천과 근거
5. **기한** — 언제까지 결정이 필요한가
6. **영향** — 결정이 지연되면 어떤 영향이 있는가

### 규칙

- 선택지를 제시할 때 장단점을 함께 명시한다
- 결정이 내려지면 `/decision`으로 기록한다
- 긴급도를 명시한다 (긴급 / 일반 / 참고)

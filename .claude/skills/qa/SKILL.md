# QA 프로세스 실행

서비스의 품질을 검증한다. 4단계 프로세스로 진행하며, QA 최종 판정권을 보유한다.

## 작업 기준

### 4단계 프로세스

1. **테스트 계획** — 기능 명세 기반 테스트 케이스 작성
2. **테스트 실행** — 기능, UI, API, 엣지 케이스 테스트
3. **버그 리포트** — 재현 경로, 심각도, 스크린샷 포함
4. **최종 판정** — Pass / Fail / Conditional Pass

### 판정 기준

- **Pass** — Critical·Major 버그 0건
- **Conditional Pass** — Minor 버그만 존재, 출시 후 패치 가능
- **Fail** — Critical 또는 Major 버그 존재
